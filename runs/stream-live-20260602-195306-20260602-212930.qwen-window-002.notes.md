*   **用户流程：** 首页浏览视频片段 -> 点击“我要出演” -> 选择角色（男主/女主） -> 上传照片 -> 生成。
*   **计费与状态逻辑：** 生成时扣除积分/钻。需判断余额：若足够直接扣费；若不足触发充值页面。由于算力消耗大，生成需要排队，个人主页需展示排队进度（如“排到第几百几十几”）。

*   **技术预处理（关键）：** 用户上传照片后，后台需进行复杂预处理：
    *   **去刘海：** 检测并去除刘海（讲师指出目前效果仍有瑕疵）。
    *   **闭嘴处理：** 检测是否露牙/张嘴，强制模型将嘴闭上，以保证换脸一致性。
    *   **逐帧替换：** 视频打散为帧，识别每一帧的目标人脸进行移植。
*   **开发建议：** 后端极其复杂，不建议学员自行开发后端，仅做APP端前端/UI交互即可作为简历亮点（多模态/视频处理经验）。

*   **实现方式：** 调用模型接口（如扣子应用），非桌面应用。风格固定，无需多样化。

**3. AI产品经理转行指南（后半段）：**
*   **核心观点：** 不要沉迷于具体插件/工具的学习（变化太快），要掌握底层能力（业务链路、成本效率分析）。
*   **面试痛点：** 面试官问“怎么理解业务”，候选人常背诵公司百科，而面试官想听的是“谁在付出代价”、“效率卡点在哪里”。
*   **学习路线（3个月降维打击）：**
    *   **前提：** 每天雷打不动2小时，年龄45岁以下。
    *   **第一阶段：** 大模型原理（入门钥匙）。
    *   **第二阶段：** RAG（检索增强生成），重点攻克知识库搭建与检索策略，解决幻觉问题。复刻开源项目。
    *   **第三阶段：** Agent（智能体），掌握感知、记忆、规划、工具调用。
    *   **第四阶段：** 商业落地，模型选型与效果评估。
