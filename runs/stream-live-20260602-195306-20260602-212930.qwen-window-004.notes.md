## Faithful Notes
本窗口主要演示了使用 **Hyper Frame**（基于 Remotion）结合 AI Agent（如 Cursor/Claude Code/Kimi）进行视频生成的完整工作流，并详细拆解了接单平台上的相关任务类型与报价。

1.  **Hyper Frame 原理与操作逻辑**：
    *   Hyper Frame 的本质是“用代码写网页”。视频画面是一个网页容器，动画、字幕、音频播放均通过网页技术实现。
    *   操作流程：将长文本（如小红书帖子）粘贴给 Agent -> Agent 生成初始项目代码 -> 在浏览器中预览（网页形式） -> 通过对话调整细节（字体大小、横竖屏、字幕对齐等） -> Agent 修改底层代码 -> 最终导出 MP4。
    *   讲师强调这是一个“扣令”（Prompt Engineering/Debugging）的过程，需要不断挑毛病（如字幕不全、字号太小）。

2.  **多项目管理与工具切换**：
    *   每个视频对应一个独立的本地文件夹（Project）。
    *   演示了在 Cursor/Trae 中通过命令行或 UI 切换不同项目文件夹的操作。
    *   提到 DeepSeek API 额度问题（个人额度用完），转而测试 Kimi 2.6 和 Claude Code。
    *   指出 Kimi 2.6 在处理字幕精简时出现逻辑错误（将18条字幕强行精简为7条，导致音画不同步），认为其能力下降。

3.  **接单任务详解（重点）**：
    *   **任务类型 A：Hyper Frame 批量视频制作**。
        *   内容：将提供的文字素材转化为视频。
        *   数量：几十条到大几十条。
        *   单价：**100元/条**。
        *   要求：需区分横屏/竖屏，调整时长，处理细节。
        *   交付物：MP4 视频文件。
        *   风险提示：严禁直接交付 Demo 或重复内容，客户上传会判重。
    *   **任务类型 B：高难度/混剪类视频**。
        *   内容：基于已有的爆款视频链接进行“改头换面”，保持 80-90% 还原度但需有差异。
        *   数量：约 6 条。
        *   单价：**300元/条**。
        *   工具：不限（可用 Hyper Frame，也可用剪映、Code 编程等），重点在于结果。
        *   案例参考：讲师展示了一个“2026年5月27日开始学习AI”的推广视频作为样例。

4.  **工具推荐与对比**：
    *   **Claude Code**：被讲师誉为“目前最强的通用 Agent”，写代码和办公都很好用，强烈推荐。
    *   **Kimi 2.6**：演示中表现不佳，字幕处理逻辑混乱。
    *   **本地模型（Qwen 3.5B等）**：不推荐用于此类交付，因为显存小、不稳定，且无法保证服务器级稳定性。
    *   **剪映**：虽然可用，但在处理大量标准化单子时效率不如 AI Coding 自动化方案。
