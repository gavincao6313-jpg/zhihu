# 基于 Obsidian + Claude Code 构建个人第二大脑

## 1. 内容概览

本视频演示了如何利用 **Obsidian**（作为本地知识库）和 **Claude Code**（作为 AI Agent）结合 **“卡巴西的知识库方法”**（Karpathy's Knowledge Base Method），构建一个自动化的个人第二大脑。

核心流程包括：
1.  **环境搭建**：安装 Obsidian，配置 iCloud 文件夹，并在 VS Code (Code) 中安装 `claude-code` CLI 及相关插件。
2.  **知识库初始化**：通过终端运行命令，让 AI 学习特定的 Skill（技能/提示词），对本地文件夹内的所有原始资料进行深度处理，生成实体、摘要、索引及文档间的链接图谱。
3.  **内容捕获与同步**：利用 **Obsidian Web** 浏览器插件，一键将网页文章或视频（含字幕）转换为 Markdown 笔记并添加到 Obsidian 库中。
4.  **增量更新**：通过终端指令让 AI 自动读取新添加的文件，并将其整合进现有的知识图谱中。

## 3. 关键事实

*   **核心理念**：数据离线存储，永远属于用户；利用 AI 降低阅读和整理门槛。
*   **核心工具组合**：
    *   **Obsidian**：作为本地 Markdown 笔记存储引擎，支持双向链接和图谱视图。
    *   **Claude Code**：作为 AI 代理，负责理解、分类、索引和连接文档。
    *   **Karpathy's Method**：一种构建知识库的方法论（视频中提及参考了 Andrej Karpathy 的相关思路/Skill）。
*   **关键技术点**：
    *   **Altthink**：在 Prompt 结尾添加此关键词，旨在激发 AI 的高强度思考模式。
    *   **Ingest 流程**：新文件放入目录 -> 终端通知 AI -> AI 读取 Skill -> 生成/更新 Markdown -> 融入知识图谱。
    *   **Web 插件自动化**：Obsidian Web 插件实现了从“浏览器阅读”到“笔记库收藏”的无缝衔接，特别是针对视频的**字幕提取**功能。
*   **目录结构建议**：视频中提到了创建 `ki` (Knowledge Index?) 等文件夹来存放整理后的文件，但主要逻辑是将所有原始资料放入一个总文件夹，由 AI 进行内部管理和索引。
