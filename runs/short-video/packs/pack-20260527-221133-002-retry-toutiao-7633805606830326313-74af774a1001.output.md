# Claude Code 最佳实践：仓库创作者的 82 条实战经验

## 1. 内容概览
本视频基于 GitHub 上由 Claude Code 作者 Boris Cherry 推荐的 `claude-code-best-practice` 仓库，总结了 82 条经过社区验证的实战经验。内容涵盖目录结构规范、上下文管理（防止“上下文腐烂”）、自动命令复用、多智能体并行工作流、云端自动化任务及代码审查等核心主题。旨在帮助开发者从入门到进阶，高效利用 Claude Code 提升编程效率。

## 2. 时间线

### [00:00:00 - 00:01:00] 仓库介绍与目录结构规范
介绍来源为 GitHub Trending #1 的 `claude-code-best-practice` 仓库，收录了作者持续更新的技巧及社区最佳实践。重点讲解第一条经验：目录结构设计。建议将通用规范放在根目录，模块专用规则放在子目录，团队共享规则放在全局，个人规则放在 `.clinerules`。强调通过自下而上访问加则干放个人共享跟目录全局纠正更新来优化配置。

### [00:01:00 - 00:02:00] 上下文管理与独立窗口
讲解第二条经验：上下文管理，即防止“上下文腐烂”。当上下文达到 30万-40万 token 时，模型注意力分散，老内容干扰当前任务。建议定期清空或压缩上下文。核心原理是使用独立的上下文窗口（如创建只读 agent），让不同任务分支互不干扰，实现类似人类团队的交叉自查，提高可靠性。

### [00:02:00 - 00:03:00] 自动命令与权限策略
讲解第三条经验：使用 `/com` (Composer) 自动斜杠命令作为复用的提示模板，适用于一天做两次以上的事。支持带上下文的文件夹，包含脚本、素材和数据，并能设置触发条件。第四条经验涉及权限管理：开启 Auto-approve 模式时，安全分类器会自动判断读写文件操作；对于反复确认的危险操作，建议加入白名单以减少打断。

### [00:03:00 - 00:04:00] 多智能体并行与远程控制
讲解第五条经验：并行才是王道。Boris 同时开 5 个 Claude 终端，每个 Agent 有颜色标识和任务状态。建议分工明确（如前端、后端、日志分析）。第六条经验提到 Teleport 功能，可将云端会话拉到本地，或通过手机控制本地会话。第七条经验介绍定时任务 `/loop`，可在本地最长运行 7 天，或在云端（Anthropic 服务器）运行，适合代码审查和 PR 提交，即使电脑关闭也不影响。

### [00:04:00 - 00:05:24] 代码审查与工作流总结
讲解第九条经验：代码审查。Claude Code 内置多智能体 PR 审查，能发现肉眼忽略的 bug。第十条经验：工作流实践。Boris 一天提 141 个 PR，其中 118 行保持小儿专注。手机端也可写代码。最后总结仓库全貌：包含 75 个内置命令、60 多个设置项详解、175 个环境变量等。推荐搜索 `claude-code-best-practice` 获取完整地图。
