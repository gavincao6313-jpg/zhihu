## 2. 核心知识字典（Glossary）

- **GPU (Graphics Processing Unit):** 图形处理单元，最初用于加速图形渲染，后因其并行计算能力被广泛应用于AI计算。
- **GPGPU (General-Purpose computing on Graphics Processing Units):** 利用GPU进行通用计算，即不限于图形处理，而是将GPU作为并行处理器来执行各种计算任务。
- **CUDA (Compute Unified Device Architecture):** 英伟达推出的并行计算平台和编程模型，允许开发者使用C/C++等高级语言直接在GPU上进行通用计算，是英伟达构建其AI生态的核心。
- **ASIC (Application-Specific Integrated Circuit):** 专用集成电路，为特定应用（如神经网络运算）专门设计和制造的芯片，具有高效率和低能耗，但通用性差，难以适应新算法。
- **SIMT (Single Instruction, Multiple Threads):** 单指令多线程，GPU采用的一种并行执行模式，同一个指令可以同时在多个线程上执行，每个线程可以处理不同的数据，且每个线程可以有自己的判断逻辑，提高了灵活性和能效。

- **核心论点：** GPU加速AI计算的本质在于其并行计算能力，区别于CPU的串行计算，且GPU的架构设计使其计算单元占比远高于CPU。
- **详细展开：** AI计算的本质是加速计算，而CPU的串行计算通过指令级并行提升能力已走到尽头，2015年后CPU性能每年仅提升3%，性能翻倍需20年。GPU加速AI计算的本质是并行计算。通过对比CPU和GPU的芯片架构图，CPU的计算单元（绿色部分）占比非常少，而GPU绝大多数晶体管用于计算。
  - **CPU串行计算：** 像“全能大厨”，什么都能干但效率低，强调并发（Concurrency），单位时间内只有一个任务执行。
  - **GPU并行计算：** 像“大量炒菜能手”，短时间内可有更多厨师炒菜，强调并行（Parallelism），同一时间多个运算单元同时执行任务。

### [00:31:45 - 00:57:56] GPU的前世今生：从图形学到算力系统

- **核心论点：** GPU从图形处理单元发展为AI计算的核心，经历了显卡、GPU、GPGPU、GPU集群和算力系统时代，英伟达通过持续的技术创新和生态建设，最终实现了对整个算力系统的闭环控制。
- **详细展开：**
  - **GPU的起源与图形学：** GPU的名字源于“图形处理单元”，其发展与计算机图形学密切相关。计算机图形学始于1962年，3D图形由无数个三角形组成，通过一系列矩阵运算（模型变换、世界变换、观察变换、投影变换）最终映射到屏幕像素点。顶点着色器和像素着色器是核心，分别处理点的位置和颜色光照。
  - **显卡时代 (1993-1999)：** 英伟达成立于1993年。早期英伟达采用方形构图以减少算力消耗，但与主流（Direct3D、OpenGL）不兼容，导致产品滞销。在世嘉的帮助下，英伟达调整策略，于1997年推出三角形构图显卡，并于1998年推出TNT系列。与主要竞争对手3DFX缠斗后，英伟达胜出。
  - **GPU时代 (1999年，提出GPU概念)：** 1999年，英伟达首次提出GPU概念，并发布GeForce 256，将坐标转换和光源处理从CPU转移到GPU。2000年，英伟达收购3DFX，市场仅剩英伟达和ATI（后被AMD收购）两强。
  - **GPGPU时代 (2003年，提出GPGPU概念)：** 2003年，SIGGRAPH大会首次提出GPGPU（通用计算GPU）概念，学术界开始关注GPU的并行计算能力。2005年，微软Xbox 360引入统一着色架构，预示着运算单元的统一。
  - **CUDA与Tesla架构 (2006年)：** 2006年，英伟达推出CUDA平台和Tesla架构，实现了GPU的可编程性，允许程序员用C语言调用GPU并行计算能力，极大地降低了开发难度。同年，ATI被AMD收购，形成英伟达与AMD两强格局。
  - **Fermi架构 (2010年)：** 2010年，英伟达推出Fermi架构，这是其首款真正的并行计算处理器，从并行处理器的角度兼顾图形处理，标志着GPU从专用走向通用。
  - **AI的引爆点 (AlexNet和ChatGPT)：**
    - 2009年，吴恩达发表论文指出GPU算力可将AI训练时间从几周缩短到几小时。
    - 2012年，AlexNet比赛中，Hinton和Ilya用4颗英伟达GPU一周内将图像识别成功率从74%提升到85%，而谷歌用16000颗CPU和1000台计算机才达到类似效果，这引爆了GPU在深度学习领域的应用。
    - 2018年，Transformer算法开启了Scaling Law时代。
    - 2022年，ChatGPT横空出世，进一步推动了芯片军备竞赛。
  - **GPU集群时代 (NVLink+NVSwitch)：** 单个GPU已无法满足计算需求，GPU之间互联成为瓶颈（PCI总线）。英伟达通过收购和研发推出了NVLink（总线和通信协议）和NVSwitch（专用互联芯片），大幅提升了卡间通信带宽（H100可达900GB/s），绕过了CPU。英伟达开始推出自己的AI服务器DGX系列。

### [01:15:04 - 01:21:42] AI芯片的架构之争：通用与专用的平衡

- **核心论点：** AI芯片架构主要分为SIMD、SIMT和ASIC，其中SIMT（英伟达CUDA）通过通用性实现了生态的成功，而ASIC虽然高效但通用性差，难以适应快速变化的AI算法。
- **详细展开：** 国产芯片厂商在架构选择上存在差异，主要有以下几类：
  1. **SIMD (Single Instruction, Multiple Data) 架构：** 单指令多数据，一条指令控制多个处理单元对不同数据执行相同操作。所有运算单元永远执行相同步骤，灵活性差，能耗高。
  2. **SIMT (Single Instruction, Multiple Threads) 架构：** 单指令多线程，结合SIMD和多线程处理特点，每个线程可有自己的判断逻辑，运算完成后可停止工作，降低能耗。英伟达采用此架构，对应细粒度指令集。
  3. **ASIC (Application-Specific Integrated Circuit) 架构 / DSA (Domain-Specific Architecture)：** 专用集成电路，为特定任务（如神经网络运算）专门设计和制造，如谷歌的TPU和苹果的NPU。高效、低能耗，但专用性导致与主流生态兼容性差，难以适应新算子。谷歌也大量采购英伟达GPU以弥补TPU的兼容性不足。
  英伟达的成长史是从专用到通用架构的历史，通用性意味着生态。CUDA的成功在于其SIMT架构掩盖了流水编排和并行指令的复杂性，为开发者提供了极大的便利。SIMT架构虽然不一定是最优的，但实践证明它是目前最优且长期占据主流。
  - **细粒度指令集（SIMT）：** 像乐高积木，可以组合成任何形状，适应性强。
  - **粗粒度指令集（ASIC）：** 像预制房子，高效但缺乏灵活性，难以满足新算法需求。
  ASIC架构的缺点在于软件和硬件不解耦，开发新算子需要硬件工程师协助，适配成本极高。

### [01:21:42 - 01:30:34] AI芯片的产业生态与优质算力标准

- **核心论点：** 芯片的成功不仅在于硬件设计，更在于构建完整的软件生态。选择优质AI算力需综合考虑通用性、易用性、有效性和成本。
- **详细展开：**
  - **芯片内部生态：** 一个芯片企业需要构建完整的软件生态，包括驱动（Driver）、运行时（Runtime）、编译器（Compiler）、性能分析器（Profiler）、程序调试器（Debugger）等工具，以方便开发者调用底层算力。
  - **芯片外部生态：** 需要兼容主流的AI框架和库，如CUDA生态、TVM、PyTorch、DeepSpeed、Hugging Face等。芯片厂商需要做大量的适配工作，适配效率取决于架构基因。
  - **优质算力的选择标准：**
    1. **通用性：** 芯片底层实现能接纳各种计算和数据类型，抽象程度高，能适应未来模型规模变化。
    2. **易用性：** 开发工具对编程友好，最好能与CUDA C兼容，降低代码迁移成本，沿用CUDA C的编程习惯。这是拥抱主流生态，再发展自身不可替代能力的关键。
    3. **有效性：** 算力有效利用率，英伟达GPU通常为40%-50%。更先进的架构能提高利用率，意味着用更少成本获得更多算力，体现了芯片厂商的调度能力和架构合理性。
