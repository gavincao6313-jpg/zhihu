## 2. 核心知识字典（Glossary）

*   **Prompt (提示词):** 指向大型语言模型提供的文本指令或输入，用于引导模型生成特定类型的响应。根据其在系统中的角色，可分为系统提示词（System Prompt）和用户提示词（User Prompt）。
*   **System Prompt (系统提示词):** 通常用于提供全局的任务指令或限制条件，告诉模型如何理解和生成响应。在产品开发中，它定义了AI助手的角色、行为规范等。
*   **User Prompt (用户提示词):** 通常是用户直接提供的问题或请求，系统会在特定回合中将其视为生成响应的核心输入。
*   **In-Context Learning (上下文学习):** 一种通过在Prompt中提供参考资料、示例（样板）或指令，来引导大型语言模型在不进行模型微调的情况下，提高其在特定任务上的表现能力的方法。
*   **Zero-Shot (零样本):** 在Prompt中不提供任何示例，直接要求模型完成任务。
*   **One-Shot (单样本):** 在Prompt中提供一个示例，以帮助模型理解任务并生成响应。
*   **Few-Shot (少样本):** 在Prompt中提供多个示例（多于一个），以进一步提高模型在任务上的表现。
*   **Token (令牌):** 大型语言模型处理文本的基本单位，可以是单词、子词或字符。模型的上下文窗口大小通常以Token数量衡量。
*   **Context Window Size (上下文窗口宽度):** 指大型语言模型在一次处理中能够接收和理解的Token数量上限。超出此限制的输入内容可能会被截断。

- **核心论点：** 阐述系统提示词和用户提示词在功能上的区别，以及模型在处理它们时的内部机制和上下文窗口限制。

### [00:24:41 - 00:30:00] RAG (Retrieval-Augmented Generation) 详解
- **核心论点：** RAG（检索增强生成）是为了解决大型语言模型上下文窗口有限的问题，通过在需要时从外部知识库检索信息，并将其与用户Prompt一起输入模型，从而提高模型回答的准确性和广度。
- **详细展开：**
    - **In-Context Learning的局限性：** 将参考资料和样例直接放在Prompt中（In-Context Learning）可以提高模型表现，但Prompt的上下文窗口长度是有限的，无法将所有企业知识都塞入Prompt。
    - **RAG的必要性：** 为了克服这一限制，需要一个外部的“知识库”。当模型需要特定信息时，它会去知识库中查找有用的信息，然后将这些信息与用户Prompt结合起来，再进行生成。
    - **RAG的两个核心步骤：**
        1.  **构建可检索的知识库：**
            -   **知识整理：** 以文件格式（PDF, Word, PPT, Excel）为主，也包括网页等非结构化数据。
            -   **数据清理及格式化：** 将不同格式的数据内容提取为纯文本，去除图片、表格等非文本信息。
            -   **内容切分：** 将文本内容按段落、主题或逻辑单元切分成较小的知识片段（Chunk），以便于检索。
            -   **向量化：** 将每个知识片段转化为向量（Embedding），通过Embedding接口实现。
            -   **关联元数据：** 给每个向量关联相关元数据（如文档名称、创建时间、作者、来源等），以便于检索和过滤。
            -   **载入向量数据库：** 将向量数据载入专门的向量数据库（如FAISS, Pinecone, Weaviate），并建立索引。
            -   **部署集成：** 将向量数据库集成到AI产品流程中，形成模型与知识库的搭配使用。
        2.  **模型调用知识库完成用户任务：** 模型需要“知道”何时应该从知识库中检索信息，何时可以直接回答。

## 2. 核心知识字典（Glossary）

*   **RAG (Retrieval-Augmented Generation):** 一种结合检索和生成的技术，通过从外部知识库中检索相关信息来增强大型语言模型（LLM）的生成能力，使其回答更准确、更具时效性。
*   **知识切分 (Chunking):** 将原始文档或文本数据分割成更小、更易于检索和处理的片段（chunks）的过程。切分策略对RAG的检索效果至关重要。
*   **Prompt Engineering:** 设计和优化给LLM的指令（prompt），以引导其生成所需输出的过程。在RAG中，Prompt通常包含系统指令、用户问题和检索到的相关上下文。
*   **Feu-shot (少样本学习):** 通过提供少量示例（exemplars）来指导模型学习特定任务或行为，而无需进行大量微调。在RAG中，可以通过在Prompt中加入问答示例来模拟。
*   **多模态模型 (Multimodal Model):** 能够处理和理解多种类型数据（如文本、图像、音频）的模型。在RAG中，可用于处理包含图片或表格的文档。

    2.  **意图识别：** 模型首先识别用户请求的核心任务或意图。

### [10:27 - 12:49] Coze作为AI产品经理工具的优势与Dify对比
- **核心论点：** Coze简化了AI Agent的开发流程，使产品经理能更快地将想法转化为可交互的Demo和工作流，减少传统文档和原型图的依赖。与Dify相比，Coze是商业产品，Dify是开源产品，两者在私有化部署和功能丰富度上各有侧重。
- **详细展开：** 讲师认为Coze对于AI产品经理来说非常常用，因为它改变了传统的工作方式。过去需要写大量文档、画流程图和原型图，现在可以直接在Coze上将文档转化为可运行的Demo，将流程图转化为“工作流”。工作流功能将在下周一的课程中详细讲解，它能帮助制作更复杂、更高级的智能体。讲师指出，这种可视化、低代码的开发方式使得传统的设计工具（如线框图、原型图）可能变得不再必要，因为直接制作可交互的Demo更高效。

        - **插件/工作流**: 讲师认为Coze在插件和工作流的丰富度及易用性上表现不错。

### [12:49 - 23:28] RAG知识库的深度构建与优化
- **核心论点：** RAG知识库的构建和优化是提升LLM性能的关键，包括知识整理、数据清洗、内容切分、向量化、关联元数据、载入向量数据库和部署集成。私有化部署能提供更高的可控性和数据安全性。
- **详细展开：** 讲师详细讲解了RAG的七个核心步骤：
    1.  **构建可检索的知识库：** 以文件格式存储（pdf, word, ppt, excel为主）。
    2.  **数据清理及格式化：** 将不同格式的数据内容提取为纯文本。
    3.  **内容切分（Chunk）：** 将文本内容按段落、主题或逻辑单元切分成较小的知识片段。讲师强调了“多粒度混合切分”的重要性，即根据内容特点使用不同大小的Chunk，而不是固定长度。
    4.  **向量化：** 将每个知识片段转化为向量（如OpenAI的Embedding接口）。
    5.  **关联元数据：** 给每个向量关联相关元数据（如文档名称、创建时间、作者、来源等），这对于后续的检索和过滤非常重要。
    6.  **载入向量数据库并建立索引：** 将向量数据载入如FAISS、Pinecone、Weaviate等向量数据库。
    7.  **部署集成：** 将向量数据库集成到AI产品流程中，生成模型搭配使用。
- **私有化部署的优势：** 讲师强调，如果企业对数据安全和性能有高要求，选择私有化部署RAG（如使用Dify或自研）能提供更高的可控性。你可以自由选择模型、优化切分策略、定制检索逻辑（如检索相邻片段），从而突破平台限制，实现更好的性能。
- **重要金句/原话：** “你完全可以不用Coze这些平台，你自己就做知识库，就是你的最终的向量其实是存在一些这样的向量数据库里面。这几个向量数据库都是公认的比较好用的一些向量数据库。也就是说，你可以完全脱离Coze或者是Dify这样的平台，你完全自己开发属于自己公司的RAG。”
