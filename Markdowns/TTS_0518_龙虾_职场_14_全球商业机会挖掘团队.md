# 龙虾_职场_14_全球商业机会挖掘团队

## 1. 视频元数据
- **推测主题：** 讲解如何利用OpenClaw平台搭建一个全球商业机会挖掘团队，通过数字员工实现自动化数据监控、智能商业价值判断和快速响应，从而实现“睡觉时也在自动套利”。
- **核心关键词：** OpenClaw, 数字员工, 商业机会挖掘, 自动化监控, 竞品监控, 商业检漏, 订单执行, AI分析, 定时任务, 飞书推送, 财富先机, 自动套利
- **适用受众/场景：** 适用于希望通过自动化工具提升商业决策效率、实时监控市场动态、发现并抓住商业机会的个人或企业，特别是电商从业者、房产投资者等。

## 2. 核心知识字典（Glossary）
- **全球商业机会挖掘团队：** 一个由多个数字员工组成的自动化数据监控团队，旨在通过实时监控、智能分析和快速响应，帮助用户发现并抓住全球范围内的商业机会。
- **商业检漏虾：** 专注于全网商业检漏机会的扫描与监控，捕捉低价资产、恐慌抛售等瞬时商业机会的数字员工。
- **竞品监控虾：** 专注于竞争对手动态的实时监控，追踪竞品价格、用户评价、新品上架等关键指标的数字员工。
- **订单执行虾：** 专注于订单的自动执行操作，根据竞品监控虾的指令快速完成商品购买的数字员工，是竞品监控虾的子Agent。
- **OpenClaw：** 一个平台，用于创建和管理数字员工（Agent），支持定时任务、多机器人协作、文件存储和消息推送等功能。

### [00:01:17 - 00:05:16] 解决痛点
- **核心论点：** 传统商业机会挖掘面临信息滞后、决策效率低、响应速度慢和缺乏系统性等痛点，而自动化团队可以有效解决这些问题。
- **详细展开：**
  1.  **信息滞后：** 以房地产为例，人工查询信息可能导致错过最佳时机，如房价已降但未及时得知。
  2.  **决策效率低：** 人工筛选商业信息并判断价值容易遗漏瞬时机会，无法全面掌控市场动态。
  3.  **响应速度慢：** 发现机会后无法快速行动，犹豫不决导致错失良机。
  4.  **缺乏系统性：** 商业信息来源分散，人工汇总、清洗、加工和分类耗时耗力。自动化团队可以同时监控多个渠道（如网站、电商平台），并建立自定义分析体系（如商品价格、评论监控），从而解决这些问题。例如，通过监控竞争对手的价格和用户评价，可以动态调整自身商品策略。

### [00:14:42 - 00:58:57] 打造与使用
- **核心论点：** 详细演示如何在OpenClaw平台创建并配置“全球商业机会挖掘团队”的三个数字员工，包括Agent创建、机器人配置、群组设置、角色定义、工具安装和路径调整，并进行实际测试。
- **详细展开：**
  1.  **创建OpenClaw Agent：**
      -   在OpenClaw平台创建三个Agent：`商业检漏虾`、`竞品监控虾`、`订单执行虾`。
      -   配置Agent时，模型和授权选择`NOW`，通道也选择`NOW`，后续手动配置。
  2.  **配置机器人：**
      -   在飞书（或类似平台）创建两个机器人，分别绑定`商业检漏虾`和`竞品监控虾`。
      -   配置`config.json`文件，添加`multi_robot_account`，包含两个机器人的`app_id`和`app_secret`。
      -   设置`dm_follows`为`true`，确保机器人具备交互能力。
      -   配置`bindings`，将机器人与对应的Agent ID绑定。
  3.  **创建群组并添加机器人：**
      -   创建一个名为“全球商业机会挖掘团队”的群组。
      -   将`商业检漏虾`和`竞品监控虾`两个机器人拉入群组。
      -   获取群组的`chat_id`，用于后续消息推送配置。
  4.  **配置Agent角色与工具：**
      -   **商业检漏虾：**
          -   配置`identity.md`，定义其核心定位和工作流程（每分钟定时任务，模拟数据，生产环境需对接真实API）。
          -   安装所需的工具（`secure`），并确保工具路径正确。
          -   配置消息推送的`chat_id`，确保能将监控结果推送到群组。
          -   保存监控数据和分析结果到指定JSON文件路径。
      -   **竞品监控虾：**
          -   配置`identity.md`，定义其职责（监控竞争对手动态）。
          -   安装所需的工具（`secure`），并确保工具路径正确。
          -   配置消息推送的`chat_id`。
          -   保存监控数据和分析结果到指定JSON文件路径。
          -   **重要：** 在竞品监控虾的配置中，添加`sub_agents`字段，将`订单执行虾`的Agent ID配置为其子Agent，建立层级关系。
      -   **订单执行虾：**
          -   配置`identity.md`，定义其职责（自动执行订单）。
          -   安装所需的工具（`secure`），并确保工具路径正确。
          -   配置消息推送的`chat_id`。
          -   保存订单数据到指定JSON文件路径。
  5.  **测试与问题排查：**
      -   在群组中与机器人互动，验证功能是否正常。
      -   测试过程中遇到平台bug（如文件删除后未同步、软链失效），讲师演示了如何通过`remove`指令和重新创建来解决。
      -   飞书推送问题：最初商业检漏虾推送失败（状态400，提示权限配置问题），后在未明确操作下自行恢复正常。
      -   成功演示了商业检漏虾监控房产信息（北京宋家庄房产情况），并推送了“挖到宝”的低价房产信息（挂牌价385万，历史均价520万）。
      -   成功演示了竞品监控虾监控商品（腾海玩具中毛绒玩具情况），并自动触发订单执行虾下单购买低价商品，生成了多个订单文件，并计算了预期利润（25%）。
