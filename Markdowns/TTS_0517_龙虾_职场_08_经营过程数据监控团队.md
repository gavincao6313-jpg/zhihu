- **推测主题：** 使用 OpenCloud 平台构建一个企业团队出差/项目预算超支数据分析与监控系统，实现预算分析、过程监控、支出预警及信息分发（邮件/电话）的自动化流程。

## 2. 核心知识字典（Glossary）
- **Agent (智能体):** 在 OpenCloud 平台中，Agent 是具有特定能力和职责的独立模块，可以接收请求、执行任务、与其他 Agent 协作，共同完成复杂业务流程。视频中创建了调度中心、预算分析、过程监控、支出告警、信息分发等 Agent。
- **调度中心 (Scheduler Agent):** 负责接收用户请求，根据请求内容将任务分发给相应的子 Agent，并编排整个业务流程的执行顺序。它是整个系统的核心协调者。
- **飞书表格 (Feishu Sheet):** 作为数据存储和交互的载体，用于记录团队出差、项目等预算信息和实际支出数据。系统会定时读取和更新飞书表格中的数据。
- **Skills (技能):** Agent 具备的特定功能或操作，通过 `.md` 文件定义，包含描述、调用方式、触发条件和模板等。例如，预算分析 Agent 具有“预算计算”的技能，信息分发 Agent 具有“打电话”和“发邮件”的技能。
- **定时任务 (Scheduled Task):** 在 OpenCloud 平台中配置的周期性任务，例如每分钟读取飞书表格数据，检查是否有超支情况，从而触发后续的监控和告警流程。
