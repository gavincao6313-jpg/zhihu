# 扣子（Coze）智能客服工作流实操全指南

## 1. 视频元数据
- **推测主题：** 基于扣子（Coze）平台的智能客服工作流搭建与AI编排思维实战
- **核心关键词：** Coze/扣子, 工作流(Workflow), Web Coding, 意图识别, 节点编排, 知识库检索优化, 数据预处理
- **适用受众/场景：** AI产品经理、开发者、业务运营人员；适用于企业级智能客服、售后系统、售前推荐等复杂业务逻辑的AI应用落地。

## 2. 核心知识字典（Glossary）
- **工作流 (Workflow)：** 将业务逻辑拆解为独立节点（如意图识别、数据库查询、条件判断等）并通过连线定义数据流向的确定性AI应用构建方式。优势在于高确定性、精准维护、全程可观测，适合生产环境。
- **Web Coding：** 通过自然语言直接描述需求，由AI一次性生成应用原型的开发模式。优势在于迭代极快、门槛低、表达灵活，适合项目早期快速试错与Demo验证。
- **意图识别颗粒度：** 意图分类的粗细程度。设计原则并非“越细越好”，而是必须与下游处理流程对齐。下游流程一致的意图应合并，流程不同的才拆分，以避免分类准确率下降。
- **编排者思维 (Orchestrator Mindset)：** 从“向AI提问的使用者”转变为“定义AI协作关系的设计者”。如同乐团指挥，不亲自演奏乐器，而是规定每个节点（乐手）的职责、触发时机与数据流向。
- **数据预处理节点（分析师节点）：** 在知识库检索前插入的大模型节点，用于将用户口语化、模糊的自然语言转化为标准化、高匹配度的检索关键词，显著提升知识库召回率。
- **渠道共享 (Channel Sharing)：** 数据库配置选项。在Coze中查询数据时必须开启，否则数据会被隔离在特定渠道内导致查询失败。
- **输出节点 (Output Node)：** 用于在工作流运行中途与用户进行交互的节点，区别于仅在流程结束时返回结果的“结束节点”，支持多轮对话与中间状态反馈。

### [00:00:00 - 00:22:00] 理论基石：工作流与Web Coding的互补及AI编排思维转变

**分支四（闲聊兜底）：** 设定食品客服人设，限制范围并引导回商品推荐。
调试强调：每完成一个节点单独测试，连线完整后整体试运行。输出节点用于流程中途交互，非仅流程结束。
