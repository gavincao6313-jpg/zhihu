## 1. 视频元数据
- **推测主题：** 讲解大型语言模型（LLM）的微调（Fine-tuning）技术，特别是参数高效微调（PEFT）方法，并通过酒店聊天机器人和招聘助手案例对比不同微调策略的效果。
- **核心关键词：** Fine-tuning, 微调, PEFT, LoRA, Prompt Tuning, P-Tuning, Prefix-Tuning, 函数调用, Chatbot, LLM, 大模型, 幻觉, RLHF, Qwen, Llama3, GLM4
- **适用受众/场景：** 针对希望理解和应用LLM微调技术的开发者、研究人员和工程师，解决LLM在特定任务上的性能优化和避免“幻觉”问题。

## 2. 核心知识字典（Glossary）

*   **Fine-tuning (微调):** 对预训练的大型语言模型进行进一步训练，以使其适应特定任务或数据集的过程。
*   **Parameter-Efficient Fine-Tuning (PEFT - 参数高效微调):** 一类旨在通过仅更新模型参数的一小部分来降低微调大型语言模型的计算成本和内存占用的技术。
*   **LoRA (Low-Rank Adaptation - 低秩适应微调):** 一种特定的PEFT技术，通过在预训练模型的现有权重矩阵中注入小的、低秩的矩阵，从而实现高效微调，仅训练这些新的、较小的矩阵。
*   **Prompt Tuning:** 一种PEFT方法，其中将少量可学习的“软提示”token添加到输入序列的嵌入层，在微调期间仅更新这些提示token，而基础模型的参数保持冻结。
*   **P-Tuning:** 一种PEFT方法，它使用一个小型神经网络（如LSTM或MLP）来生成连续的提示嵌入，然后将其添加到输入嵌入中。这些生成的提示嵌入是可学习的。
*   **Prefix-Tuning:** 一种PEFT方法，它将可学习的前缀向量注入到Transformer模型每个注意力机制层的输入中，而不仅仅是初始输入层。
*   **Function Call (函数调用):** 大型语言模型理解用户自然语言请求、提取结构化参数，然后调用外部函数或API（例如，查询数据库）以完成请求的能力。
*   **Hallucination (胡说八道):** 大型语言模型生成听起来合理但事实不准确或捏造的信息的现象，通常在没有底层数据支持的情况下自信地给出。


    ```
    工作流程
    1. 接单：骑手通过外卖平台（如美团、饿了么）接到顾客的订单，通常这些订单会显示餐品信息、商家位置、顾客位置等。
    2. 取餐：骑手根据平台提供的导航信息到达商家处取餐，取餐时需要确保餐品完整无损。
    3. 配送：取餐后，骑手会根据最优路线（通常由平台算法提供）将餐食送到顾客手中。骑手需要在规定的时间内完成配送。
    4. 送达：抵达顾客指定地点后，骑手将餐品交给顾客，确认完成订单。

    工作特点
    1. 时间压力：外卖骑手的工作常常需要在较短时间内完成配送，尤其是在高峰期如午餐和晚餐时间。这要求骑手具备良好的时间管理和快速应变能力。
    2. 天气挑战：无论是炎热的夏天、寒冷的冬天，还是下雨下雪天，外卖骑手都需要出门工作，气候对他们来说是一个很大的挑战。
    3. 收入结构：骑手的收入主要来源于订单的配送费用和顾客的额外小费，有时候还会平台奖励和补贴。订单量、配送距离、天气情况等都会影响收入。
    4. 灵活性：外卖骑手通常是兼职或自由职业者，他们可以选择自己的工作时间。这种工作方式为许多人提供了灵活的收入来源，尤其是需要兼顾其他工作的兼职人员。

    工作要求
    1. 熟练使用导航软件：外卖骑手需要依赖导航软件来规划路线和找到顾客的准确地址，因此对导航工具的使用要求较高。
    ```

### [12:49:00 - 18:46:00] Fine-tuning 的具体方法：全模型微调、冻结部分参数微调、轻量化微调（PEFT）

- **核心论点：** 介绍了几种主要的微调方法，重点强调了参数高效微调（PEFT）的重要性，并详细解释了Prompt Tuning、P-Tuning和Prefix-Tuning的原理。
- **核心论点：** PEFT方法通过注入少量可学习参数或利用低秩适应，显著降低了微调的计算和存储成本，特别适用于处理大规模预训练模型。
- **详细展开：**
    - **1. 全模型微调 (Full Fine-Tuning):**
        - 描述：更新模型中的所有参数。
        - 缺点：计算成本和存储成本高昂，需要大量计算资源。
    - **2. 冻结部分参数微调 (Partial Fine-Tuning / Layer Freezing):**
        - 描述：冻结模型的部分层（例如，Transformer的底层），只微调其他层（例如，顶层）。
        - 优点：相比全模型微调，计算和存储成本降低。
        - 缺点：选择冻结哪些层和微调哪些层需要经验和实验。
    - **3. 轻量化微调 (Parameter-Efficient Fine-Tuning, PEFT):**
        - 描述：在不改变大多数模型参数的情况下，添加一些低秩调整参数。这是目前最常用的微调技术。
        - 优点：显著降低训练时的计算和存储成本，特别适合处理大规模预训练模型。
        - 缺点：可能在某些复杂任务上表现不如全量微调。
        - **3.1 Prompt Tuning:**
            - 描述：在模型的输入嵌入层前添加少量可学习的“软提示”（soft prompts）。这些软提示是随机初始化的，并在微调过程中进行优化，而基础模型的参数保持冻结。
            - **视觉/屏幕内容：**
                - 图示显示一个Transformer模型。下方是输入序列，分为两部分：左侧是`prompt`（红色和黄色圆点），右侧是原始输入`I like fruits.`（蓝色圆点）。
                - 箭头从`prompt`和`I like fruits.`指向Transformer的输入层。
                - 红色箭头从`prompt`指向Transformer内部，表示`prompt`参数是可学习的。
            - **详细展开：** 演讲者解释，Prompt Tuning的思路是，不直接修改预训练模型的参数，而是在输入序列前添加一些额外的、可学习的“提示”token。这些提示token的嵌入向量是随机初始化的，然后在微调过程中进行训练，以引导模型更好地完成特定任务。模型在处理输入时，会将这些提示嵌入与原始输入嵌入拼接起来。
        - **3.2 P-Tuning:**
            - 描述：P-Tuning是Prompt Tuning的改进版。它不直接使用离散的提示token，而是使用一个小型神经网络（如LSTM或MLP）来生成连续的提示嵌入。这些生成的提示嵌入是可学习的，并被添加到原始输入嵌入中。

                - 红色箭头标记“LSTM / MLP”和“Prompt Embedding”为可学习参数。
            - **详细展开：** 演讲者解释，P-Tuning认为直接添加离散的提示token可能过于粗糙。它引入了一个小型生成网络（如LSTM或MLP）来生成更复杂的、连续的提示嵌入。这些生成的提示嵌入与原始输入嵌入拼接后，再输入到Transformer中。这种方法增加了提示的表达能力。
        - **3.3 Prefix-Tuning:**
            - 描述：Prefix-Tuning与Prompt Tuning和P-Tuning不同，它不是只在输入层添加提示，而是将可学习的“前缀”（prefix）向量注入到Transformer模型每一层的注意力机制的输入中。
            - **视觉/屏幕内容：**

## 2. 核心知识字典（Glossary）

*   **LoRA (Low-Rank Adaptation)**: 一种轻量级微调技术，通过在预训练模型的核心权重矩阵旁边注入小的、可训练的低秩矩阵（Delta W），来适应新任务。原始模型参数在微调过程中保持不变，只训练这些注入的低秩矩阵，从而显著减少了可训练参数的数量。
*   **Q Weights (Wq), K Weights (Wk), V Weights (Wv), Output Weights (Wo), Feed-Forward Weights (W1, W2)**: Transformer模型中关键的权重矩阵，用于计算Query、Key、Value向量，以及在多头注意力机制和前馈网络中的线性变换。这些矩阵的尺寸决定了模型的主要参数量。
*   **矩阵加法 (Matrix Addition)**: 两个相同尺寸的矩阵进行元素级别的相加，结果是一个与原矩阵尺寸相同的新矩阵。在LoRA中，它用于将微调后的Delta W矩阵与原始的预训练权重矩阵W相加。
*   **矩阵乘法 (Matrix Multiplication)**: 两个矩阵相乘，其结果矩阵的尺寸由第一个矩阵的行数和第二个矩阵的列数决定，且第一个矩阵的列数必须等于第二个矩阵的行数。在LoRA中，Delta W矩阵被分解为两个低秩矩阵A和B的乘积（Delta W = A * B），以减少可训练参数。
*   **QLoRA (Quantized Low-Rank Adaptation)**: 在LoRA的基础上，进一步优化显存占用的技术。它将预训练模型的大部分参数量化为4-bit精度（例如，4-bit NormalFloat），而LoRA适配器（A和B矩阵）则以更高精度（如bfloat16）进行训练。这使得在单张消费级GPU上微调大型模型成为可能。

### [16:58:00 - 24:00:00] LoRA的参数量优势与QLoRA的显存优化

## 2. 核心知识字典（Glossary）
- **QLoRA (Quantized Low-Rank Adaptation):** 一种通过量化技术（如将32位浮点数参数量化为8位整数）来显著减少大型语言模型微调时显存占用的方法，同时保持可接受的模型性能。它本质上是LoRA的一种优化实现。
- **过拟合 (Overfitting):** 模型在训练数据上表现良好，但在未见过的新数据（验证集或测试集）上表现不佳的现象。这通常是由于模型过度学习了训练数据中的噪声和特定模式，而未能捕捉到数据的普遍规律。
- **欠拟合 (Underfitting):** 模型在训练数据和新数据上都表现不佳的现象。这通常是由于模型复杂度不足，无法从数据中学习到足够的特征和模式。
- **训练集 (Training Set):** 用于模型实际训练和学习的数据集，通常占总数据的70-80%。
- **验证集 (Validation Set):** 用于在模型训练过程中评估模型表现的数据集，以便随时调整超参数（如学习率、batch size等），通常占总数据的10-15%。
- **测试集 (Test Set):** 用于在模型训练和调优完成后，对模型最终性能进行评估的数据集，通常占总数据的10-15%。

## 2. 核心知识字典（Glossary）

*   **Fine-tuning (微调):** 在已经训练好的模型基础上，进一步调整，让模型的输出能更符合预期。它是一个训练过程，与预训练类似，但通常使用更小的数据集和更低的学习率。
*   **LoRA (Low-Rank Adaptation):** 一种轻量化微调技术，通过将预训练模型参数矩阵分解为低秩矩阵进行微调，从而减少了需要优化的参数数量。它的思想是在不改变大多数模型参数的情况下，添加一些低秩调整参数。
*   **PEFT (Parameter-Efficient Fine-Tuning):** 参数高效微调技术的总称，旨在通过只微调模型参数的一个小子集来解决全量微调的计算和存储成本问题。LoRA、Prompt Tuning、P-Tuning、Prefix Tuning 都是 PEFT 的具体方法。
*   **自主可控:** 在中国语境下，特指技术、产品、系统等从核心技术到生产制造全链条的自主研发和控制，避免受制于人。在模型选择上，这意味着即使是预训练模型，也倾向于选择国内自主研发的模型，而非国外模型。
*   **早停 (Early Stopping):** 在模型训练过程中，通过监控验证集上的性能（如损失函数），当验证集性能连续几个 epoch 不再提升甚至开始下降时，提前停止训练，以防止模型过拟合。

### [00:00:00 - 00:50:00] LoRA 参数计算与可训练参数

*   **核心论点：** LoRA 微调通过引入低秩矩阵来调整预训练模型的权重，显著减少了可训练参数的数量，从而降低了微调的计算成本。
*   **详细展开：**
    *   在 Transformer 架构中，每一层都有多个参数矩阵（如 Wq, Wk, Wv, Wo, W1, W2）。例如，一个 12289 * 12288 的矩阵（Wq, Wk, Wv）在 96 层中，每层有 6 套这样的矩阵。
    *   LoRA 的核心思想是，不直接修改原始的权重矩阵 W，而是引入一个低秩的更新矩阵 ΔW。这个 ΔW 又可以分解为两个更小的矩阵 A 和 B 的乘积（A 是 12289 * r 的矩阵，B 是 r * 12288 的矩阵），其中 r 是一个远小于原始矩阵维度的秩（通常取值如 4, 8, 16, 32）。
    *   通过这种方式，原始的权重矩阵 W 加上 ΔW，实际上只训练了 A 和 B 两个小矩阵的参数，而不是整个 W 矩阵的参数。
    *   以一个 70 亿参数的模型为例，其总参数量是 70 亿，但通过 LoRA 微调，可训练的参数数量会远小于 70 亿，这大大降低了微调所需的计算资源。
    *   演讲者强调，面试中可能会问到如何计算一个 7B 模型进行 LoRA 微调时，总共有多少个可训练参数，这需要掌握具体的计算方法。
*   **视觉/屏幕内容：** 屏幕上显示了标题“低秩适应微调（Low-Rank Adaptation, LoRA）”，以及 LoRA 的数学表达式和矩阵维度示例：
    *   `Wq, Wk, Wv: 12289 * 12288 * 128 (每层96套，共96层)`
    *   `Wo: 12289 * 12288 (96层)`
    *   `W1, W2: 12289 * 12288 * 4 (96层)`
    *   `Wq + ΔW (ΔW是12289 * 12288的矩阵)`
    *   `Wq + A * B (A是12289 * r 的矩阵，B是r * 12288 的矩阵)`
    *   `r 比较常见的取值为: 4、8、16、32`
*   **重要金句/原话：** “你选定了一个模型之后，你其实能算得出来你的 LoRA 参数里面总共有多少个可训练参数，大家注意啊，是可训练参数。”

### [10:05:00 - 11:29:00] 轻量化微调技术 (PEFT) 的具体方法

*   **核心论点：** Prompt Tuning、P-Tuning 和 Prefix Tuning 是三种轻量化微调技术，它们通过在模型外部或内部注入少量可训练参数来引导模型行为，而无需修改大部分预训练模型权重。
*   **详细展开：**
    *   **Prompt Tuning:** 在用户输入的 Prompt 前面添加一些可训练的参数（prompt embedding），这些参数与原始输入一起进入 Transformer 模型。模型本身的参数不改变。
    *   **P-Tuning:** 类似于 Prompt Tuning，也是在输入 Embedding 层面进行操作，但可能通过一个小的 LSTM/MLP 网络来生成 Prompt Embedding，使其更具表达力。同样，Transformer Blocks 的参数保持不变。
    *   **Prefix Tuning:** 在 Transformer 模型的每一层（或部分层）的输入中，注入可训练的 Prefix 参数。这意味着它在模型内部的每一层都添加了少量参数，而不是仅仅在输入层。
    *   这些方法的核心优势在于，它们只训练少量参数，大大降低了微调的计算成本和存储需求，同时在许多任务上能达到接近全量微调的效果。
*   **视觉/屏幕内容：** 屏幕上显示了标题“轻量化微调（Parameter-Efficient Fine-tuning, PEFT）”，并详细展示了 Prompt Tuning、P-Tuning 和 Prefix Tuning 的架构图。
    *   **Prompt Tuning 图示:** 显示 `prompt`（红色方块）在 `I like fruits.`（蓝色方块）之前输入 `Transformer` 模型。
    *   **P-Tuning 图示:** 显示 `LSTM / MLP` 和 `Prompt Embedding` 结合生成 `Prompt Encoder`，然后与 `Embedding` 一起输入 `Transformer Blocks`。
    *   **Prefix Tuning 图示:** 显示 `prefix`（红色方块）在 `Transformer` 模型的每一层输入中被注入，与 `I like fruits.`（蓝色方块）一起处理。
*   **重要金句/原话：** “Prompt Tuning 和 P-Tuning 都没有在模型里面加参数。它在用户问的问题那个 Prompt 前面，那个 Prompt 比如说你说了四个字，它在你这四个字前面又加了一组参数。”

### [12:50:00 - 13:50:00] Prompt Tuning 与模型参数

*   **核心论点：** Prompt Tuning 和 P-Tuning 不改变模型参数，而是在用户输入的 Prompt 前面添加可训练参数。LoRA 和 Prefix Tuning 则是在模型内部（Transformer 层）添加参数。
*   **详细展开：**
    *   **Prompt Tuning / P-Tuning:** 这两种技术的核心在于，它们不修改预训练模型的任何权重。它们只是在用户输入的 Prompt 前面（或通过一个小的编码器）添加一些可训练的向量（Prompt Embedding），这些向量与原始输入一起进入模型。因此，它们是在模型“之外”添加参数。
    *   **LoRA / Prefix Tuning:** 这两种技术则是在 Transformer 模型的内部结构中添加参数。LoRA 通过低秩分解在权重矩阵中添加小的更新矩阵。Prefix Tuning 则是在 Transformer 的每一层（或部分层）的输入中注入可训练的 Prefix 参数。
    *   演讲者强调，理解这些技术在何处添加参数是关键。
*   **视觉/屏幕内容：** 屏幕上显示了标题“轻量化微调（Parameter-Efficient Fine-tuning, PEFT）”，并再次展示了 Prompt Tuning、P-Tuning 和 Prefix Tuning 的架构图。演讲者用红色圈出了 `Transformer Blocks` 和 `Prompt Encoder` 部分，强调参数添加的位置。
*   **重要金句/原话：** “Prompt Tuning 和 P-Tuning 都没有在模型里面加参数。它在用户问的问题那个 Prompt 前面加参数。”

### [13:50:00 - 14:20:00] 模型理解能力与微调

*   **核心论点：** 当基座模型无法区分或理解特定概念时，即使是人类专家看来天差地别的做法，模型也可能认为“都很好”，此时微调是必要的。
*   **详细展开：**
    *   演讲者举例说明，在某些任务中，模型可能无法区分两种截然不同的处理方式，即使在专业人士看来，这两种方式有“天壤之别”。
    *   模型之所以做不好任务，可能是因为它压根理解不了这些概念之间的差异。
    *   微调的目的就是去优化 Transformer 模型理解特定事物的能力，让它能够区分这些重要的概念，从而提高其处理任务的能力。
*   **重要金句/原话：** “你找一个模型，比如说 GPT 也好或者什么也好，你让它在你的场景里面去做任务，你发现它做不好。你就会就着你的任务，给它出一道选择题，让它去区分一个概念。比如说，刚才我们讲的，一个店员跟顾客的对话，同样的场景下，另外一个店员跟顾客也发生了对话，然后来处理一个比较复杂棘手的问题。那你把两个店员的这个处理方式拿出来，你问一个模型，请问它这两个店员的处理方式和回应，谁处理得好？你就会发现有的时候，它做不好这个任务，是因为它压根理解不了这两种做法是不一样的。”
