# Claude Code 进阶指南：解决联网、图片识别及上下文监控问题

## 1. 内容概览

本期视频主要解决在使用 **Claude Code (CC)** 时遇到的常见痛点，包括无法联网搜索、无法识别图片、无法阅读网页以及无法监控上下文进度和 Plan 使用情况。

核心解决方案分为两部分：
1.  **功能扩展（MCP 技术）**：通过安装特定的 MCP (Model Context Protocol) 工具，赋予 AI 使用外部软件的能力，实现联网搜索、网页读取和图片识别。
2.  **插件改造（上下文监控）**：针对非原生支持的国产套餐，利用 CC 的高自由度修改开源插件代码，将数据源替换为国产套餐接口，并保留原有的进度条 UI 样式。

## 2. 时间线

### [00:00:00 - 00:01:00] 痛点与 MCP 概念引入

*   **常见问题**：上期分享了 CC 的安装和技巧，但用户反馈存在局限：没法联网搜索、没法识别图片、没法阅读网页、没法监控上下文进度和 plan 使用情况。
*   **解决方案核心**：MCP (Model Context Protocol)。
*   **MCP 定义**：AI 以前局限于自身知识库，有了 MCP 就具备了使用外部软件的能力。它是 AI 和外部软件之间的一座桥梁。

### [00:01:00 - 00:02:00] 推荐三个免费开源 MCP 方案

为了分别解决上述问题，推荐以下三个主流且免费的开源方案：
1.  **联网搜索**：推荐使用 `brave-search-mcp-server`。
    *   **优势**：拥有独立的搜索索引，对 AI 的读取特别友好。
2.  **网页读取**：推荐使用 `firecrawl-mcp-server`。
    *   **优势**：能过滤掉网页里的杂乱元素，只把最核心的代码和文字扒下来给 AI。
3.  **图片识别**：推荐使用 `screenshot-mcp`。
    *   **优势**：能自己截取当前画面或指定窗口去分析报错原因，让 AI 看图排错。

### [00:02:00 - 00:03:00] MCP 安装演示 (以 Brave Search 为例)

*   **前置条件**：需要获取一个 Brave Search 的 API Key。
*   **安装步骤**：
    1.  打开终端。
    2.  粘贴安装命令（需替换为你获取的 API Key）。
    3.  回车自动运行。
*   **其他工具**：另外两个工具（Firecrawl 和 Screenshot）直接复制对应的命令即可。
*   **总结**：MCP 工具的好处在于轻量和即插即用。GitHub 上有大量好用的 MCP 工具可供探索。

### [00:03:00 - 00:04:00] 上下文监控与插件改造

*   **原生插件问题**：有一个开源插件可以在输入框下方以进度条形式直观显示上下文用量，但该插件仅支持 Claude Cloud 订阅套餐。
*   **改造思路**：
    1.  **原理**：CC 终端版自由度高，插件是开源项目，可以修改代码。
    2.  **机制对比**：作者发现其使用的“国产套餐”机制与 Cloud 套餐机制类似（都有总量限制和每小时限制）。
    3.  **执行步骤**：
        *   第一步：让 CC 搜索如何获取国产套餐的数据。
        *   第二步：让 CC 写脚本，将国产套餐的数据读取代码替换到插件中，并设置每 5 分钟刷新一次。
*   **UI 修复**：
    *   **问题**：替换后，监控从“进度条形式”变成了“文本形式”。
    *   **解决**：利用自然语言告诉 CC，既然数据来源换了，样式应该不动。要求保留原本的进度条显示颜色和搭配。
    *   **结果**：CC 很快帮作者改好了代码，实现了用国产套餐数据驱动原生的进度条 UI。
*   **核心观点**：了解原理（前端、后端、套餐机制）才能进行这种替换。多动手尝试，提升知识储备。
