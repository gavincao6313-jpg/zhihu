# FineTuning_04_【神秘嘉宾】大模型时代的AI产品新挑战

## 1. 视频元数据
- **推测主题：** 本视频深入探讨了大模型时代AI产品开发面临的新挑战与机遇，强调了产品经理应如何转变思维模式，理解大模型的核心能力与局限性，并识别未来AI产品落地的关键方向和技术栈。
- **核心关键词：** 大模型, AI产品, AI Agent, 泛化性, RLHF, 模型幻觉, 多模态, 一站式交互, PMF, TPF, AI原生应用, MaaS, 中间件, 应用层, 智能涌现, 自监督学习, 思维惯性。
- **适用受众/场景：** 适用于AI产品经理、创业者、技术研发人员以及任何对大模型时代AI产品发展趋势、落地策略和职业机会感兴趣的人士，旨在帮助他们跳出传统思维，抓住AI 2.0时代的变革机遇。

## 2. 核心知识字典（Glossary）

-   **AI Agent (智能体)**: [00:01:56] 大模型是其“大脑”，但真正将大模型转化为一个可用的“机器人”或执行者，需要依靠AI Agent来实现规划、调度和工具使用等能力。
-   **泛化性 (Generalization)**: [00:15:29] 指模型将所学知识应用于新的、未见过的情况的能力。在视频中，通过火车票预订的例子解释，即模型在学习了火车票预订流程后，能举一反三地理解飞机票、轮船票预订的类似逻辑，而无需重新训练。这是大模型相较于小模型最大的突破点。
-   **RLHF (Reinforcement Learning from Human Feedback，基于人类反馈的强化学习)**: [00:17:23] 一种训练大模型的方法。传统方法依赖人工标注数据来纠正模型错误，而RLHF通过模型自身生成内容、自我评估并反馈给模型进行迭代训练，实现了自动化学习和优化，大幅提升了模型效果。
-   **模型幻觉 (Model Hallucination)**: [00:21:34] 指大模型在生成内容时一本正经地“胡说八道”，提供事实性错误或不准确的信息。这是自回归语言模型固有的缺陷，目前尚无明确的根本性解决方案。
-   **一站式交互 (One-stop Interaction)**: [01:07:03] 大模型时代人机交互的未来模式。用户只需发出一个指令（可以是语音或文字），AI系统就能自动串联并完成整个流程，无需用户进行多步操作或在不同应用间切换。
-   **PMF (Product-Market Fit，产品市场匹配)**: [00:56:09] 传统产品开发中，产品经理首先考虑产品与市场需求的匹配度。
-   **TPF (Technology-Product Fit，技术产品匹配)**: [00:56:12] 大模型时代新的产品开发出发点，产品经理应首先考虑技术与现有产品的匹配，即如何利用AI技术升级现有产品，而非从零开始寻找新市场。
-   **AI原生应用 (AI-Native Application)**: [00:57:18] 李彦宏提出的概念，指完全基于AI能力构建的应用，其特点是自然语言交互、充分利用大模型理解/生成/推理能力，且交互层级不超过两级菜单。
-   **MaaS (Model-as-a-Service，模型即服务)**: [01:27:46] 介于底层基础模型和中间件之间的一层。它提供对基础模型的微调、调优、数据标注等服务，使通用大模型能在特定垂直领域表现更好。
-   **中间件 (Middleware)**: [01:28:13] 位于MaaS层之上，负责组装AI Agent，将定制化的大模型能力与外部工具、系统进行连接，实现更复杂的任务规划和执行。

### [00:11:59 - 00:25:25] LLMs带来的变化与存在的缺陷

-   **核心论点：** 大模型并非技术革命，其核心突破在于泛化性的大幅提升，但仍存在模型幻觉等固有缺陷，未来可能走向自监督学习和世界模型。
-   **详细展开：**
    -   **LLMs并非技术革命：**
        -   严格意义上，以ChatGPT为代表的LLMs并非技术革命。其底层神经网络架构Transformer早在2017年就已出现。预训练模型GPT系列（GPT-1, GPT-2, GPT-3）也并非2022年才出现。
        -   真正的突破在于**泛化性（Generalization）**的大幅提升。
        -   泛化性提升的原因：早期猜测是RLHF（基于人类反馈的强化学习）和预训练数据中包含代码。
        -   RLHF的突破：OpenAI首次成功应用RLHF，用少量数据（InstructGPT Paper中SFT监督微调数据1.3K，RM奖励模型数据33K，总计5万多条数据）就训练出显著效果，这在以前的大模型时代是不可想象的，因为人工清洗百万级数据成本极高。
        -   **泛化性解释：** 举例说明，传统小模型在学习了火车票预订后，无法举一反三地处理飞机票预订；而大模型在学习了火车票预订和“火车、飞机、轮船都是出行工具”后，能自动推断飞机票预订的流程，体现了强大的举一反三能力。
        -   RLHF机制：模型自动生成数据，自我打分（通过奖励模型），然后反馈给模型进行训练，使其趋向于生成高奖励的答案。
    -   **LLMs存在能力缺陷：**
        -   **模型幻觉（Hallucination）：** 自回归语言模型存在“一本正经胡说八道”的问题，即事实错误。这是因为其本质是概率模型，每次预测都会有微小错误，随着预测次数增多，错误概率会指数级放大。
        -   引用Meta AI首席科学家Yann LeCun的观点：当前的自回归语言模型（LLM）是两三年前的技术，未来五年内可能被淘汰，下一个时代将是**自监督学习（Self-supervised Learning）**的天下。
        -   Yann LeCun提出的公式：`P(正确概率) = (1 - 错误概率)^N`，其中N是预测次数。N越大，正确概率越低。
        -   未来AI应具备的能力：推理能力、复杂计划和分层规划能力（世界模型），而当前大模型缺乏这些能力，这也是AI Agent落地的重要性。
-   **视觉/屏幕内容：**
    -   **[00:12:04] 幻灯片标题：** “LLMs带来了哪些变化”
        -   左侧：**LLMs并非技术革命**
            -   使用Transformer神经网络架构 (Transformer, 2017)。
            -   预训练模型GPT (GPT-1, GPT-2, GPT-3)。

            -   自回归语言模型存在无明确解决方案的能力缺陷。
            -   **现在：** 模型幻觉 (一本正经的胡说八道)。
            -   `P(正确概率) = (1 - 错误概率)^N` (N=预测的次数)。
            -   **未来：** self-supervised learning? 推理能力, 复杂计划和分层规划。

### [00:25:26 - 00:30:02] LLMs带来的变化：单模态与多模态

-   **核心论点：** 单模态大模型主要提升现有场景的效果上限，难以创造新场景；多模态AI则有望改变人机交互方式，创造新的场景和机会，是真正的未来。
-   **详细展开：**
    -   **单模态（Single-modality）：**
        -   主要提升了单模态模型的**效果上限**，例如在NLP领域，客服的智能程度从80%提升到更高。
        -   解决了快语问题、一语多义、理解上限、专有名词等问题。
        -   **难以创造新场景**：单模态大模型更多是在原有场景中提升效率和智能，而非开辟全新领域。
        -   案例：微博内容审核事件。用户发布“日葛福岛当年最大港口爆发抗议集会”被屏蔽，原因是系统将“日葛福岛”误识别为敏感词。这反映了上一代模型在理解上的局限性。大模型若能跨越语义鸿沟，此类问题将减少。
    -   **多模态（Multi-modality）：**
        -   **改变人与计算机的交互方式**，大概率会产生新场景、新形态和新机会。
        -   从电影走向现实：例如《钢铁侠》中的贾维斯（Jarvis）、电影《Her》中的虚拟伙伴等。
        -   讲师认为，多模态才是真正的未来，前提是AI Agent等技术能够真正落地。

### [00:30:03 - 00:55:51] 大模型的落地场景与产品思维转变

-   **核心论点：** 大模型短期内最容易落地的是已渗透的场景，且需要从“变化”而非“惯性思维”出发做产品，理解真实需求，并利用AI打破信息交互时间差，实现“一站式交互”。
-   **详细展开：**
    -   **大模型的需求场景：**
        -   AI 1.0已渗透大部分领域，但效果有限。大模型最突出的落地机会在于已渗透的场景中，通过提升效果带来变革。
        -   参考艾瑞咨询2020年的报告，人工智能渗透的主要环节中，“人机对话”渗透度最高，覆盖所有领域。因此，2023年出现了大量人机对话产品（如百度文心一言、科大讯飞星火、字节豆包等）。
        -   **产品思维：** 识别AI 1.0渗透深、成熟度高的行业，这些领域在大模型时代变革机会越大。
    -   **案例一：肥皂工厂的空盒检测 (00:34:12 - 00:37:12)**
        -   **问题：** 自动化流水线后期，肥皂盒可能为空，需人工取出，成本高。
        -   **传统解决方案（惯性思维）：** 投入数百万开发AI视觉检测和机械臂，识别并取出空盒。
        -   **真实需求：** 确保未装肥皂的盒子不装箱。
        -   **简单解决方案：** 在流水线末端放置一个电风扇，将空盒吹走（因为空盒轻）。
        -   **启示：** 绝大部分提需求者给不出真实需求，他们提出的往往是实现需求的方式，而非真正的需求。做产品要跳出“惯性思维”，先理解真实需求，再寻找最佳解决方案。
    -   **案例二：早餐店排队问题 (00:38:44 - 00:52:58)**
        -   **背景：** 2015-2016年移动互联网刚起步，现金支付为主。早餐店早高峰排队严重，导致顾客流失，店长通过增加店员、多队并行来应对，但早高峰后人员效率低下，造成成本浪费。
        -   **真实需求：** 少花钱（降低人力成本）但多赚钱（不减少收入或增加收入）。
        -   **阻碍问题：** 队伍太长导致丢单，根本原因是单位时间内成交单处理效率太低。
        -   **解决方案（讲师实践）：** 开发微信公众号预点餐系统。顾客在排队时通过公众号提前点餐、支付，到店后直接取餐。
        -   **效果：** 售卖效率提升30%以上。麦当劳、肯德基等类似系统在半年后才出现。
        -   **细节：** 为避免取餐混乱，设计了“我要取餐”按钮，点击后订单会置顶显示在店员操作界面，方便店员快速交付。
        -   **启示：**
            -   **移动互联网时代：** 手机互联打破了物理空间的信息交互时间差（异步通信），实现了移动支付。
            -   **大模型时代：** 将从“人串联单一能力”转变为**“一站式交互”**，计算机串联全流程，用户只需发号施令。
            -   举例：识别花卉。现在需要“拿出手机 -> 打开百度 -> 选择拍照搜索 -> 对准花 -> 拍照 -> 出来一堆答案 -> 自己点进去一个一个看 -> 自己筛选答案”。未来只需“拿起一个设备（眼镜/手机/胸牌）-> 对准花 -> 问‘这幅画叫什么名字’”，AI直接给出答案。
            -   强调：做产品要思考“我们有什么”（AI的能力）和“我们需要处理什么”（用户痛点）。

### [01:11:12 - 01:27:27] LLMs存在的问题与落地关键要素

-   **核心论点：** 大模型存在模型幻觉、价值观对齐、不可解释性等问题，需要通过限制应用场景、指定可信数据源等方式规避。成功的LLM落地需兼顾实用性、体验性和安全性。
-   **详细展开：**
    -   **LLMs存在的问题：**
        -   **模型幻觉（Hallucination）：**
            -   表现：事实错误、一本正经胡说八道。
            -   案例：知乎上有人用大模型刷答案，大模型又通过搜索查到这些错误答案并返回给用户，形成恶性循环。
            -   规避方案：在应用层面，限制模型回答的领域（例如，客服模型只回答客服问题，其他问题一概不答）；指定可信数据源（例如，只从维基百科、百度百科等官方渠道获取信息）。
        -   **价值观对齐（Value Alignment）：**
            -   问题：不同文化、地域、人群有不同的价值观，模型难以理解和适应。
            -   案例：美国黑人文化中，送西瓜可能被视为侮辱。大模型无法理解这种文化背景。
            -   规避方案：将模型应用于特定领域，限定使用范围和人群，了解其价值观。
        -   **不可解释性（Inexplicability）：**
            -   问题：模型能表现出某种能力（如智能涌现），但无法解释其发生的原因和机制。
            -   案例：当模型参数达到11B以上时，性能指标会指数级提升，但科学家无法解释其原理，业内称之为“智能涌现”。
            -   规避方案：在产品开发中，必须通过大量实验来确定模型能力，并预留充足的实验时间。
        -   **其他问题：** 时效性（数据更新）、使用成本（算力消耗）。
    -   **LLMs落地的三个关键要素：**
        1.  **实用性（Practicality）：** 能够执行具体的任务。目前大模型在单轮对话表现良好，多轮对话仍是挑战。
        2.  **体验性（Experience）：** 能否提供类似人类的体感，如情感共鸣、人设陪伴。
        3.  **安全性（Safety）：** 生成内容是否合规合理，包括内容正确性和内容正向性。
    -   **LLMs短期落地方向：**
        -   **智能化体感明显，存在内容生产需求且风险可控的场景。**
        -   **内容生产：** 将“写作”变为“改作”，大模型在文本摘要、续写、补全、智能创作等方面表现突出。
        -   **风险可控：** 对“胡说八道”有容忍性。
            -   面向业务内部提供辅助（如GitHub Copilot，即使出错也可人工修改）。
            -   第三方内容评论（非官方形式）。
            -   娱乐场景（如游戏NPC，用户对错误有容忍度）。
        -   案例：文心一格（文生图）、GitHub Copilot（代码生成）、Midjourney（AI艺术生成）等。
    -   **LLMs中期落地方向：智能体（Agent）**
        -   Agent = 大模型 + 技艺 + 主动规划 + 工具使用。
        -   大模型本身只有理解和生成能力，缺乏认知和规划能力。Agent作为“四肢”，执行“大脑”（大模型）的指令，实现复杂任务。
        -   Agent将极大拓展多模态AI的应用空间，例如实体交互、图像/视频识别处理、调用内部系统等。

### [01:27:28 - 01:34:47] 从LLMs到可落地应用：分层架构与机会

-   **核心论点：** LLM到可落地应用将形成一个分层架构：底层模型、MaaS、中间件和应用层。其中，中间件和应用层将是未来AI从业者最大的机会所在。
-   **详细展开：**
    -   **LLM到可落地应用的分层架构：**
        1.  **底层模型（Base Model）：** 如GPT-4、文心一言、通义千问。这一层机会已不多，2023年的“千模大战”后，主要由少数大厂和头部创业公司占据。
        2.  **MaaS（Model-as-a-Service）：** 如百度文心千帆平台、字节方舟平台。基于底层大模型进行微调、调优、数据标注，使其在特定垂直领域更好用。讲师认同李彦宏观点，未来MaaS层可能也只有少数大厂提供，小公司没必要自己搞。
        3.  **中间件（Middleware）：** 如OpenAI Assistants API、百度AI原生应用工作台。这一层负责组装Agent，将定制好的大模型与外部工具、系统连接，实现规划、执行等能力。讲师认为，未来能容纳大部分AI从业者（产品、研发、测试）的机会就在这一层。
        4.  **应用（Application）：** 最终面向用户的产品。讲师认为最大的机会在应用层，就像移动互联网时代的微信、抖音等超级App。应用层是基于MaaS层定制化模型，再通过中间件进行管理和开发。
    -   **GPTs和星河社区：** OpenAI的GPTs和百度星河社区大模型工具中心，被视为介于中间件和应用层之间的产品。它们允许用户通过Prompt定义AI角色，直接调用大模型能力，但其可用场景相对有限，除非底层模型无所不知、无所不能。
    -   **就业机会：** 讲师强调，对于互联网从业人员，底层模型和MaaS层的机会有限，因为需要大量GPU算力且头部效应明显。最大的就业机会在于中间件和应用层，这里可以站在巨人的肩膀上，利用现有模型和平台，结合业务场景进行创新和开发。

-   **核心论点：** AI产品成功关键在于流量和利润，短期内应关注现有平台内的小程序或B2B提效，中期Agent是方向，长期则需持续关注技术变革。
-   **详细展开：**
    -   **什么样的AI产品容易火？**
        -   当前AI产品面临流量获取成本高的问题。
        -   成功的AI产品（如妙鸭相机）往往在现有大厂生态（如微信小程序）中孵化。
        -   未来独立APP很难成功，小而美的产品或小程序仍有机会。
        -   国内互联网流量已是存量市场，竞争激烈，需争夺用户使用时长。海外市场（如TikTok、拼多多海外版）仍有增量机会。
    -   **GPTs是否属于Agent？**
        -   GPTs更像一个简易版的智能体，通过Prompt扮演角色，但缺乏Agent的规划、调度和工具使用等核心能力。
    -   **多模态大模型应用前景？**
        -   多模态是未来，但技术尚不成熟，目前多为简单应用（如文生图）。
    -   **AI产品方向选择？**
        -   所有商业服务的核心是利润。选择方向应先看其是否有利润前景，再考虑AI如何赋能。
        -   短期内有前景的方向：客服、助手、编程辅助（如GitHub Copilot）。
        -   海外市场机会大于国内。
    -   **编程数据对大模型推理能力的影响？**
        -   加入编程数据确实能提升模型推理能力，但具体机制和效果仍需实验验证。
    -   **哪个模型能走到最后？**
        -   难以预测。建议选择大厂模型或主流开源模型进行尝试和对比。
    -   **自回归与自监督学习的区别？**
        -   Yann LeCun认为自回归模型缺乏认知、推理和规划能力，自监督学习是未来。
    -   **ChatGPT泛化性好的原因？**
        -   模型规模大、训练数据多。开源小模型（如7B、13B）是为特定垂直场景定制，无需通用泛化能力，且大厂不会开源最好的模型。
    -   **自监督学习是自己训练自己吗？**
        -   某种程度上是，模型生成答案并自我打分，然后反馈训练。但作为概率模型，仍会出错，且错误会随交互次数放大。
    -   **多模态是否是真正意义上的多模态？**
        -   讲师认为目前真正的多模态尚未完全实现，仍在探索中。
    -   **LLM落地三要素：** 实用性、体验性、安全性。
    -   **短期/中期/长期发展概念：**
        -   短期（2023-2024）：现有场景的AI赋能、内容生成。
        -   中期（1-3年）：Agent落地，实现复杂任务的规划和执行。
        -   长期：难以预测。
    -   **降低算力需求的研究进展？**
        -   讲师表示不清楚具体进展，但肯定有相关研究。
    -   **如何衡量大模型好坏？**
        -   难以用单一指标（如大标签准确率）衡量，需结合具体落地场景的效果。
    -   **中间件和Function Call的关系？**
        -   Function Call是中间件的一部分，中间件是更广义的概念，包含调用外部接口的能力。
    -   **AI产品开发中的数据隔离？**
        -   公有云平台依赖平台机制，私有化部署可自行控制。
    -   **AI取代人类？**
        -   不是AI取代人，而是掌握AI技术的人取代不掌握AI技术的人。
