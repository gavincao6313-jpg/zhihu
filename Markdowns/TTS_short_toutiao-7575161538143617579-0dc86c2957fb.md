# RAG实战：从零编写代码与基于Dify的工作流搭建

## 1. 内容概览
本视频分为两个主要部分。第一部分详细演示了如何使用Python代码从零实现RAG（检索增强生成）系统，包括环境配置、文档切块、向量化存储及检索流程。第二部分介绍了使用低代码平台Dify搭建RAG工作流的步骤，涵盖知识库导入、工作流编排（提取关键内容、知识检索、大模型回答）及测试优化。

## 3. 关键事实
*   **RAG流程**：离线链路（切片、Embedding、存储）+ 在线链路（Query改写、检索、重排序、Prompt拼接、生成）。
*   **代码环境**：使用`uv`创建虚拟环境，依赖库包括`langchain`、`chromadb`、`google-generativeai`等。
*   **向量数据库**：示例中使用ChromaDB，存储结构包含ID、Text、Embedding。
*   **Dify配置**：
    *   **知识库切片**：支持按字符数或语义分割，推荐“高质量”模式。
    *   **检索策略**：支持混合检索（关键词+向量），可配置Rerank模型提升精度。
    *   **工作流节点**：开始、提取关键内容、知识检索、大模型、结束。
*   **参数细节**：
    *   `TopK`：决定检索返回的片段数量，影响上下文长度和成本。
    *   `Temperature`：RAG中通常设为0以保证答案基于事实，减少幻觉。
    *   `Memory Window`：控制对话历史记忆的轮数。

## 4. 可检索细节
*   **Gemini API Key获取**：需在浏览器访问Google AI Studio获取，并添加到系统环境变量。
*   **UV命令**：`uv init`, `uv add <package>`, `uv run python script.py`。
*   **ChromaDB存储逻辑**：每个chunk包含`id`（标签）、`document`（文本）、`embedding`（向量）。
*   **Dify知识库设置路径**：工作室 -> 知识库 -> 创建知识库 -> 上传文件 -> 分片模式（通用/自动）-> 嵌入模型 -> 索引方式（高质量/经济）-> 混合检索开关 -> TopK设置。
*   **Dify工作流提示词模板**：
    ```markdown
    你是一个专业的客服...
    背景：...
    任务：根据以下知识库内容回答用户问题...
    知识库内容：{{knowledge_retrieval.result}}
    用户问题：{{start.question}}
    ```
*   **Rerank模型作用**：对初步检索的结果进行重排序，提高相关性分数最高的片段排在前面。
