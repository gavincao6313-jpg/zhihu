## 2. 核心知识字典（Glossary）
- **多模态大模型 (Multimodal Large Models):** 以大型语言模型（LLM）为核心，融合了视觉（图像）等其他模态信息，使其能够理解和生成跨模态内容。其核心范式是将视觉信息通过模态桥接网络作为语言模型的输入，由语言模型进行理解和生成。
- **代码大模型 (Code Large Models):** 针对编程语言和代码任务进行优化的语言模型，能够理解代码、生成代码、调试代码，并回答与代码相关的问题。它通过在大量代码数据上进行预训练和指令微调来获得这些能力。
- **封神榜 (Fengshenbang):** IDEA研究院认知计算与自然语言研究中心（CCNL）推出的开源大模型系列项目，旨在成为中文AIGC和认知智能的基础设施。该项目自2021年开源以来，已发布100多个模型，涵盖30多种主流模型结构，并在多模态、代码等领域有广泛实践。
- **指令微调 (Instruction Fine-tuning / SFT):** 一种训练大模型的方法，通过在高质量的指令-响应对数据上进行微调，使模型能够更好地理解和遵循人类指令，从而提升其在特定任务上的表现和泛化能力。
- **强化学习与人类反馈 (Reinforcement Learning from Human Feedback, RLHF):** 一种高级的微调技术，通过收集人类对模型生成结果的偏好反馈，训练一个奖励模型（Reward Model），然后利用该奖励模型通过强化学习（如PPO算法）进一步优化语言模型，使其生成更符合人类偏好和价值观的响应。

### [00:28:00 - 00:35:41] 大语言模型（LLM）浪潮与多模态新范式
- **核心论点：** 2022年11月之后，中文LLM市场百花齐放，为开发者提供了更多机会。LLM以自然语言生成下一个Token的方式，实现了世界知识的高度压缩。新的多模态范式以LLM为核心，通过模态桥接将视觉信息融入LLM，实现更深层次的理解和生成。
- **详细展开：**
    *   **LLM浪潮与开源机遇：**
        *   2022年11月之后，中国LLM市场“百花齐放”，对普通开发者和开源社区有利，因为竞争加剧促使更多公司开源模型（如Llama、智谱、百川、苏神、紫牙-封神榜）。
        *   LLM的核心能力：以自然语言生成下一个Token，实现大量世界知识的压缩。
        *   利用开源LLM的两种主要方式：
            1.  **模态融合：** 将LLM与视觉模态结合，如Ziya-Visual。
            2.  **领域迁移：** 将LLM应用于特定领域，如Ziya-Code。
    *   **多模态大模型：如何在LLM基础上融合视觉模态：**
        *   **传统多模态模型范式：** 文本输入 -> 文本编码器 -> 文本嵌入；图像输入 -> 视觉编码器 -> 视觉嵌入；然后通过“模态交互”进行融合，最终输出文本。
        *   **LLM为核心的多模态大模型新范式：** 图像输入 -> 视觉编码器 -> 视觉嵌入；视觉嵌入通过“模态桥接”网络作为“大型语言模型（LLM）”的输入，LLM再结合文本输入，最终输出文本。
            *   **核心转变：** LLM成为核心，负责理解图像在隐层空间中的意义并生成答案。
            *   **参数量：** 通常比传统多模态模型大得多。
            *   **示例：** OpenAI的DALL-E 3（ChatGPT与DALL-E结合）也采用了类似思路。
    *   **传统多模态模型示例：**
        *   **CLIP (Contrastive Language-Image Pre-training)：** OpenAI通过大量图文对进行对比学习，学习图文之间的关系，用于图像检索、图文匹配等任务。
        *   **BLIP-2：** 重要的LLM为核心的多模态模型。
            *   **第一阶段：** 训练Q-Former，通过Learned Queries对齐图文（Caption）信息。Q-Former将海量图文对压缩到Learned Queries中。
            *   **第二阶段：** 冻结LLM，通过一个Projecting层将Learned Queries连接到LLM作为输入，并进行有监督训练。
            *   **关键点：** Learned Queries包含图片信息，并转化为LLM可理解的形式。
            *   **任务：** 给定图片生成描述（Captioning）和视觉问答（VQA）。
            *   **局限性：** 性能仍有提升空间。

### [00:39:30 - 00:59:50] 多模态测评方法与Ziya-Visual的经济有效融合
- **核心论点：** 多模态模型的评估方法多样，但都存在局限性，难以完全反映生成质量。Ziya-Visual通过两阶段训练，利用高质量图文数据和LLM的知识推理能力，实现了经济有效的模态融合，并在中文多模态领域取得领先。
- **详细展开：**
    *   **多模态测评方法：**
        1.  **MME评估：**
            *   **方式：** 针对图片提问，评估模型在存在性、计数、位置、推理、数学、翻译等多个维度上的理解能力（如“是否有一只猫？”）。
            *   **优点：** 测评维度广。
            *   **缺点：** 难以反映真实的生成质量，更偏向分类任务，而非LLM擅长的生成能力。
        2.  **GPT-4评分：**
            *   **方式：** 通过文本描述图片内容（甚至包含坐标细节），然后让GPT-4评估模型针对该图片问题的回答质量。
            *   **优点：** 能够比较真实地反映生成效果。
            *   **缺点：** 受限于GPT-4自身的多模态认知范围和能力。
        3.  **传统多模态数据集评测：**
            *   **方式：** 包括图片描述（Image Captioning，评估BLEU, ROUGE等指标）、图文检索（Image-Text Retrieval）、视觉问答（VQA）。
            *   **优点：** 分数可量化。
            *   **缺点：** 难以反映真实生成质量，且构建复杂VQA数据集成本高。
    *   **Ziya-Visual：经济有效的模态融合方法：**
        *   **基础模型：** 基于自研的Ziya-LLaMA-13B-v1（已开源）和自训练的Clip模型。
        *   **两阶段训练：**
            *   **一阶段（Projection）：** 对齐文本和摘要特征。使用2000万图文匹配数据，8张A100 GPU训练7000步。仅训练Projecting层。
            *   **二阶段（Projection + Q-Former）：** 引入多样化指令对话数据（问答数据）。收集60万高质量图文问答数据，8张A100 GPU训练2天。训练Projecting层和Q-Former。
        *   **核心优势：** 引入大语言模型后，模型能理解图片并结合世界知识进行衍生生成（如识别泰坦尼克号图片后，能提供导演、上映时间、奖项等信息），这是传统模型无法做到的。
        *   **性能表现：** 在VQA Score和GPT4 Score上，Ziya-Visual在当时开源模型中表现领先（蓝色柱状图）。
            *   *GPT-4评分细节：* 在“Complace”维度上可能略低，因为Ziya-Visual回答更直接，而GPT-4有时偏好更复杂但可能无关的冗长回答。
            *   *最新对比：* Virtual CPM等模型也证实Ziya-Visual在同类模型中表现最佳。
    *   **Ziya-Visual的数据挑战与解决方案：**
        *   **挑战：**
            *   中文图文对数据稀少且质量低（图片质量差、图文匹配度低、描述多为品牌名而非内容）。
            *   中文图文问答数据简单且多样性低（多为“有多少个？”、“什么颜色？”等简单问题，缺乏深度交流）。
        *   **解决方案：**
            *   从开源数据集中进行翻译、搜集、标注。
            *   采用规则过滤和模型打分（利用泰语CLIP评估图文匹配度）。
            *   利用GPT-4生成和标注数据。
            *   **成果：** 最终获得高质量英文图文匹配数据2000万，中文1500万；高质量英文图文问答数据60万+，中文60万+。
            *   **未来方向：** 构建细粒度图文问答数据，使模型能围绕图片重点进行回答，避免“跑偏”。
    *   **多模态大模型的新价值：**
        *   **新理解-开放：** 模型能理解图片深层含义并结合世界知识进行推理。
            *   **示例1：** 分析Logo设计（识别灯泡、大脑等视觉元素，并联想到神经元、电子线路、科学技术感等抽象概念）。
            *   **示例2：** 开放定义任务（根据用户提供的规则，判断焊工操作是否规范，模型能理解“焊接”的危险性并给出推理）。
        *   **多轮对话：** 模型能基于多张图片进行连贯的多轮故事创作和总结，保持上下文一致性。

### [00:59:50 - 01:13:50] Ziya-Visual优化方向与代码大语言模型

            *   解决模型在理解细粒度视觉细节（如特定颜色、物体局部）时可能“跑偏”的问题。
            *   需要引入目标检测、分割等技术，使模型能更精确地关注图片中的关键细节。

    *   **代码大语言模型：如何在LLM上微调自己的Copilot：**
        *   **核心问题：** 如何快速提升LLM理解指令的能力，尤其是在代码领域。
        *   **“求助”策略：**
            1.  **求助开源LLM：** 选择一个好的基础模型（需要评估指标）。
            2.  **求助数据（微调数据）：** 利用ChatGPT/GPT-4作为强大的标注者，生成高质量的指令数据。
        *   **如何产生高质量的Instructions：**
            *   人工编写Prompt效率低，生成Instruction质量和多样性不高。

            *   **WizardLM (Microsoft) 方法：** 提升LLM理解复杂指令的能力。
                *   **In-Depth Evolving (深度演化)：** 通过增加约束、深化、具体化、增加推理步骤、复杂化输入等方式，使Prompt更复杂。
                *   **In-Breadth Evolving (广度演化)：** 通过变异（Mutation）生成近似Prompt，避免模型沿着单一路径越走越偏。
                *   **Elimination Evolving (终止演化)：** 设定终止条件，如ChatGPT判断无信息增益、难以生成响应、无有效响应、简单复制Prompt等。

        *   **代码上的微调方法：WizardCoder：**
            *   微软基于WizardLM对代码领域进行改进。
            *   **改进点：**
                1.  移除WizardLM中“深化、复杂化输入和广度演化”等部分。
                2.  简化并统一Prompt模板形式，便于大规模生成。
                3.  增加两个代码相关的演化指令：代码调试（code debugging）和代码时空复杂度约束（code time-space complexity constraints），例如要求生成时间复杂度更低的代码。
