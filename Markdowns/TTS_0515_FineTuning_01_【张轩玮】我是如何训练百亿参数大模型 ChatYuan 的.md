## 2. 核心知识字典（Glossary）

-   **Transformer:** 2017年Google提出的基于Attention机制的模型框架，由Encoder和Decoder两部分组成，广泛应用于NLP、CV等领域，是当前大模型的基础架构。
-   **Encoder-Decoder架构:** Transformer模型的一种经典结构，Encoder负责对输入序列进行编码，Decoder负责根据编码信息生成输出序列，适用于机器翻译等输入输出对等任务。
-   **Decoder-only架构 (如GPT):** 仅包含Transformer的Decoder部分，通过预测下一个Token进行训练，擅长文本生成任务，且在大模型规模下也能展现出较强的理解能力。
-   **BPE (Byte Pair Encoding):** 一种常用的Subword分词算法，通过迭代合并语料中出现频率最高的字节对来构建词表，旨在平衡字符级和单词级表示的优缺点，有效处理未登录词。
-   **RLHF (Reinforcement Learning from Human Feedback):** 一种强化学习方法，通过人工标注的偏好数据训练奖励模型（Reward Model），再利用奖励模型对语言模型进行强化学习，使其输出更符合人类偏好。

### [00:12:15 - 00:15:45] Transformer回顾

-   **核心论点：** 回顾Transformer模型的基本架构，强调其在NLP及其他领域的广泛应用和其Attention机制的优雅性。
-   **详细展开：**
    *   Transformer模型于2017年由Google在论文《Attention Is All You Need》中提出。
    *   该框架自提出后应用越来越广泛，不仅限于NLP，还扩展到图像视觉、视频等领域。
    *   其核心优势在于Self-Attention机制，能够优雅地提取特征和进行特征融合，且模型不依赖于RNN或CNN等传统结构，更统一和简洁。
    *   **Transformer架构组成：**
        *   **Encoder (编码器):** 位于左侧，由多层堆叠的Encoder Block组成。每个Encoder Block包含两个子层：
            1.  **Multi-Head Attention (多头注意力):** 主要是Self-Attention，允许模型同时关注输入序列的不同部分。
            2.  **Feed-Forward Network (FFN):** 一个两层的前馈神经网络。
        *   **Decoder (解码器):** 位于右侧，由多层堆叠的Decoder Block组成。每个Decoder Block包含三个子层：
            1.  **Masked Multi-Head Attention (带掩码的多头注意力):** 也是Self-Attention，但会掩盖未来信息，确保生成时只依赖于已生成的部分。
            2.  **Multi-Head Attention (多头注意力):** 这是一个Cross-Attention，其Query (Q) 来自Decoder的上一个子层输出，Key (K) 和 Value (V) 来自Encoder的输出，用于融合Encoder的上下文信息。
            3.  **Feed-Forward Network (FFN):** 一个两层的前馈神经网络。
    *   **输入部分：** Encoder和Decoder的输入都由两部分组成：
        1.  **Token Embedding (词嵌入):** 词本身的语义表示。
        2.  **Positional Encoding (位置编码):** 词在序列中的位置信息，对于序列任务至关重要。

### [00:17:10 - 00:22:00] BERT与GPT模型对比

-   **核心论点：** 对比BERT和GPT两种代表性模型，阐述它们在Transformer架构选择（Encoder-only vs. Decoder-only）和预训练目标上的差异，以及由此带来的能力侧重和发展历程。
-   **详细展开：**
    *   **BERT (Bidirectional Encoder Representations from Transformers):**
        *   2018年由Google发布，是双向预训练模型。
        *   引领了上一代大模型时代，后续有各种改进。
        *   架构上只使用了Transformer的Encoder部分。
        *   预训练目标是MLM (Masked Language Model)，即掩盖部分词语，然后通过上下文恢复被掩盖的词。
        *   擅长理解性任务，双向理解能力强。
    *   **GPT (Generative Pre-trained Transformer):**
        *   2018年由OpenAI开始研发。
        *   架构上只使用了Transformer的Decoder部分（且没有Cross-Attention，只包含Masked Multi-Head Attention和FFN）。
        *   预训练目标是预测下一个Token，无需人为掩盖词语，训练效率高。
        *   早期在理解任务上不如BERT的双向理解能力强，因此光芒一度被BERT掩盖。
        *   OpenAI坚持发展GPT系列，事实证明其选择非常明智，因为现在是“生成时代”，GPT架构在生成任务上具有天然优势。
        *   “万物皆可生成”的趋势使得GPT架构成为主流。

### [00:22:00 - 00:37:20] 模型架构-相关分析 (Encoder-Decoder vs. Decoder-only)

-   **核心论点：** 从算法和工程层面深入分析Encoder-Decoder和Decoder-only架构的优劣势及适用场景，解释为何当前大模型普遍倾向于Decoder-only架构。
-   **详细展开：**
    *   **算法层面：**
        1.  **理解能力:** Encoder（如BERT）的双向理解能力强于Decoder（如GPT），因为Encoder可以同时看到上下文信息。
        2.  **生成能力:** Decoder（如GPT）的生成能力强于Encoder，因为Decoder天然设计为循环解码生成。
        3.  **任务匹配：**
            *   **输入信息 ≈ 输出信息 (对等):** 如机器翻译，Encoder-Decoder更适合，因为它需要对输入有良好理解，同时能进行生成。这也是Transformer原始论文的任务。
            *   **输入信息 > 输出信息 (压缩):** 如分类、抽取任务，Encoder-only更适合。输入文本长，输出是有限的类别或抽取结果，信息被压缩。BERT时代多为这类任务。
            *   **输入信息 < 输出信息 (发散):** 如文本创作、生成任务，Decoder-only更适合。输入指令短，输出内容长且需要发散推理。
        4.  **生成能力差异:** Encoder-only模型（如BERT）一般认为不具备生成能力（除非进行特殊处理）。Decoder-only模型（如GPT）则具有一定的理解能力，因为可以将理解部分作为Prompt或In-context信息。
        5.  **大模型趋势:** 当Decoder-only模型规模足够大（百亿、千亿参数）时，其理解能力也能达到满意水平，使得理解能力上的微小差距不再重要。因此，Decoder-only能够同时涵盖理解和生成任务，成为主流选择。
    *   **工程层面：**
        1.  **预训练目标构建：**
            *   Encoder-Decoder模型需要对预训练数据构建目标（如掩盖词恢复），构建方法（破坏程度、频率）对效果有影响，需要人为干预。
            *   Decoder-only模型的预训练目标通常只需预测下一个Token，无需单独构建目标，更简洁高效，减少了人为干预对结果的影响。
        2.  **参数共享与扩展性：**
            *   Encoder-Decoder模型（如T5）的理解和生成是两套独立的参数，在模型规模扩展时容易遇到瓶颈。目前千亿规模的Decoder-only模型很多，但千亿规模的T5模型（Encoder-Decoder）尚未出现，Google也只做到百亿级别，说明其在Scale-up上存在挑战。
            *   Decoder-only模型只有一套参数，更容易进行缩放。
        3.  **多轮数据处理：**
            *   Encoder-Decoder处理多轮数据需要构建多份数据，对同一多轮对话进行多次Encoder输入和Decoder输出的构建。
            *   Decoder-only模型只需将多轮对话拼接成一条数据，一次性进行训练，其目标始终是预测下一个词，构建更简单。
        4.  **训练效率：** Decoder-only的训练效率更高。Encoder-Decoder需要对同一条数据构建多次不同目标进行充分训练，影响训练效率。训练一个T5模型可能需要10倍于Decoder-only模型的时间才能达到相同的Token训练量。
    *   **模型架构-生成式 (Attention Masking):**
        *   **Fully-visible (全可见):** 类似于Encoder部分，每个Token都能看到序列中的所有其他Token（包括自身、过去和未来）。图中黑色方块表示可见。
        *   **Causal (因果):** 类似于Decoder的Masked Self-Attention，每个Token只能看到自身和它之前的Token，看不到未来的Token。图中灰色方块表示可见。
        *   **Causal with prefix (带前缀的因果):** 结合了前两种，输入部分（prefix）是Fully-visible，可以双向看到；生成部分（causal）是Causal，只能看到过去。这种方式在参数量与GPT类似的情况下，能让输入部分有更好的理解能力。代表模型是UniLM。
    *   **代表性模型与Attention方式：**
        *   **T5 (Encoder-Decoder):** 完整的Transformer架构，Encoder部分是Fully-visible，Decoder部分是Causal。
        *   **GPT (Language Model):** 纯Decoder-only架构，所有层都是Causal，每个Token只能看到前面的Token。
        *   **Prefix LM (如UniLM):** 结合了Fully-visible和Causal。输入部分（X1, X2, X3）是Fully-visible，输出部分（Y1, Y2, Y3）是Causal，但Y部分可以双向看到X部分。这种架构在节省参数的同时，增强了输入部分的理解能力。

        *   还有更多开源数据集正在涌现。
    *   **清洗流程：** 收集到的数据来源多样，质量参差不齐，且可能存在重复，因此需要进行清洗。
        1.  **规则过滤 (快速过滤不必要文本):**
            *   保留完整的文本：例如，以标点符号结尾的句子，去除片段式文本。
            *   过滤重复文本：只保留连续重复文本中的一个。

            *   删除黄赌毒、偏见、煽动性文本：通过关键词过滤。
            *   去除包含前端代码的文本：爬取网页时可能带有的前端代码。
        2.  **相似去重 (避免过分训练):**
            *   数据量大，需采用快速方法，如布隆过滤器或局部哈希进行去重。
            *   重复数据会导致模型对某些Token过分训练，影响效果。
        3.  **模型过滤 (更精细化，成本更高):**
            *   **质量判别模型:** 训练一个模型来判断文本质量，或使用现有模型计算PPL (Perplexity) 等指标，过滤低质量文本。

### [00:47:15 - 00:51:15] 中文字典的构建 (BPE算法)

-   **核心论点：** 介绍使用SentencePiece工具和BPE算法构建中文字典的过程，强调Subword分词在平衡词表大小和表示能力方面的优势。
-   **详细展开：**
    *   **分词的重要性：** 对于语言模型，词的表示（Embedding）和位置信息都非常重要。
    *   **传统分词方法的局限性：**
        *   **Char-level (字符级):** 对于英文，词表太小（26个字母），区分性不足，不利于训练和理解。对于中文，字数多，但如果只用字，词的语义信息会丢失。
        *   **Word-level (单词级):** 词表太大（几百万单词），导致Embedding矩阵过大，占用显存高，训练不动。
    *   **Subword (子词) 方式：** 介于字符级和单词级之间，平衡了词表大小和表示能力，是目前主流的分词方法。BPE (Byte Pair Encoding) 是其中一种常用算法。
    *   **BPE算法流程：**
        1.  准备一个大规模训练语料。
        2.  确定期望的Subword词表大小（例如32000个Token）。
        3.  将单词拆分成最小单位（例如英文的26个字母加上标点符号，中文的单个汉字），作为初始词表。
        4.  在语料上统计词对的频率，选取出现频率最高的词对合并成新的Subword单元。
        5.  重复步骤4，直到达到设定的Subword词表大小或最高频率为1（即无法再合并）。
        6.  最终生成指定大小的词表。
    *   **工具：** SentencePiece是常用的BPE实现工具，支持多种分词算法。

### [00:51:15 - 00:57:10] 预训练目标的构建 (T5为例)

-   **核心论点：** 以T5模型为例，详细解释预训练目标（Text-to-Text）的构建方式，即通过掩盖原始文本中的词语并用特殊符号代替，然后让模型恢复这些被掩盖的词语。
-   **详细展开：**
    *   **T5预训练目标：** T5模型采用Text-to-Text的统一范式，将所有NLP任务都视为文本生成任务。其预训练目标是通过“去噪”（denoising）方式构建的。
    *   **构建过程：**
        1.  **原始文本 (Original text):** 例如 "Thank you for inviting me to your party last week."
        2.  **输入 (Inputs) 构建：** 从原始文本中随机选择一些词语或连续片段进行掩盖，并用特殊的占位符（如`<X>`, `<Y>`）代替。例如，"Thank you `<X>` me to your party `<Y>` week."
        3.  **目标 (Targets) 构建：** 目标序列是按顺序恢复被掩盖的词语，每个被掩盖的片段前加上对应的占位符，最后以结束符（`<Z>`）结尾。例如，"<X> for inviting <Y> last <Z>"。
        4.  模型的目标就是学习如何将Inputs映射到Targets，即根据上下文恢复被掩盖的词语。如果模型能很好地完成这个任务，说明它理解了文本的语义并能进行推理。
    *   **构建策略 (Mixture of Denoises):**
        *   T5论文中提出了多种去噪策略，通过调整破坏的长度和频率来构建不同的预训练目标。
        *   **R-denoiser (短窗口，低破坏频率):** 破坏的词语数量少，窗口小。
        *   **S-denoiser (长窗口，低破坏频率):** 破坏的词语数量多，窗口大，但只破坏一次，类似于将上文作为输入，下文全部掩盖。
        *   **X-denoiser (长窗口，高破坏频率):** 破坏的词语数量多，窗口大，频率高。
        *   这些策略及其组合比例对模型训练效果有显著影响。Google专门研究了不同配比的效果，并给出了经验值。实际操作中，需要通过实验找到合适的配比。

-   **详细展开：**
    *   **预训练整体流程：**
        1.  **数据清洗：** 经过规则过滤、相似去重、模型过滤后的干净数据。
        2.  **采样样本：** 根据预设的配比（基于人工经验或规则）从清洗后的数据中采样。
        3.  **目标构建：** 对每个采样数据，按照X-denoiser, R-denoiser, S-denoiser等策略进行破坏，构建Input和Target对。这些策略的配比也有经验值可参考。

    *   **训练参数和数据量的关系 (Scaling Laws):**
        *   存在一个数据量和参数量的最佳比例，并非数据量或参数量越大越好。
        *   OpenAI和DeepMind在几年前就发现了这个规律：当训练的Token数量是模型参数量的20倍时，是最佳的配比。

        *   如果Token数量大于20倍参数量，提升收益不明显，可能造成资源浪费。
        *   这个“缩放定律”可作为未来训练模型时选择参数和数据量的参考。

### [01:04:15 - 01:14:45] 预训练相关 - 并行化训练与技巧 (学习率、Batch Size、Checkpoint、位置编码)

-   **核心论点：** 介绍大模型训练中常用的并行化策略（数据并行、Tensor并行、流水线并行）以及关键训练技巧，特别是位置编码在处理长序列时的重要性。
-   **详细展开：**
    *   **并行化训练：**
        1.  **数据并行 (Data Parallelism):** 最容易理解。每个GPU复制一份完整的模型参数，然后将数据分成多份，每个GPU训练一份数据。训练速度与GPU数量成正比。
        2.  **Tensor并行 (Tensor Parallelism / 模型并行):** 在Tensor计算层面进行切分。当单个GPU无法容纳整个模型参数时，将模型内部的矩阵（Tensor）切分成小块，分发到不同的GPU上进行计算，最后再合并结果。例如，矩阵A乘以矩阵B，可以将B切成B1和B2，分别与A计算得到C1和C2，再拼接成完整的C。
        3.  **流水线并行 (Pipeline Parallelism):** 按模型层进行切分。将模型的不同层分配到不同的GPU上，形成一个计算流水线。例如，第0层在GPU0，第1层在GPU1，以此类推。数据在不同GPU之间串联流动，解决单个GPU无法承担所有模型参数的问题。
    *   **训练技巧：**
        1.  **学习率 (Learning Rate):**
            *   前期线性增加 (Warmup)，后期逐渐衰减 (Decay)。
            *   前期加速学习，更快接近最优点；后期减小调整幅度，避免跳出最优点，保持稳定。
            *   现代优化器（如Adam）通常已内置这些调度策略，只需设置初始学习率。
        2.  **Batch Size (批次大小):**
            *   开始时使用较小Batch Size，然后逐渐增大。
            *   在GPU显存允许的情况下，Batch Size应开到最大，以提高训练效率和稳定性。
            *   前期小Batch Size有助于模型参数的展开和探索，后期大Batch Size能更好地拟合数据。
        3.  **定期保存Checkpoint (检查点):**
            *   大模型预训练耗时久，可能出现意外（节点断开、硬件故障）。
            *   定期保存模型检查点（例如每天保存一次），可以在中断后从最近的检查点继续训练，避免从头开始，节省时间和成本。
            *   保存频率需权衡存储空间和保存耗时。
        4.  **位置嵌入 (Positional Embedding):**
            *   随着模型上下文长度需求增加（从512/1024到3068/8192甚至100k+），位置编码成为关键研究方向。
            *   **可学习的位置编码:** 训练时多长，推理时就只能多长，超出长度会产生不可预测的错误。
            *   **相对位置编码:** 更受青睐，如ALiBi (Attention with Linear Biases) 和 RoPE (Rotary Positional Embedding)。
            *   **ALiBi原理:** 在Query (Q) 和 Key (K) 的点积结果上，加上一个与相对位置相关的偏置矩阵。距离越近的Token，偏置值越大，反之越小，从而引入相对位置信息。其优势在于无需可训练参数，推理时可以扩展到比训练时更长的序列长度。

        *   **数据并行:** 多个GPU复制模型，每个GPU处理一部分数据。
        *   **Tensor并行:** 矩阵乘法A*B=C，将B切分B1, B2，分别计算A*B1=C1, A*B2=C2，再合并C1, C2得到C。

-   **核心论点：** 总结了ChatYuan训练中的关键技术，包括混合预训练、混合采样、分阶段训练、增强语言模型和Encoder/Decoder能力，以及重新构建中文字典，并介绍了PromptCLUE作为ChatYuan的前身。

        *   **大规模预训练：** 使用数百GB中文语料，训练了100万步，累积训练了1.5万亿个中文词语Token。

        *   **混合预训练：** 将下游任务数据作为预训练语料的一部分，与通用预训练语料一起训练，减少任务灾难遗忘，缩小预训练与下游任务的距离，更好地适应下游任务。
        *   **混合采样：** 针对众多数据量差异大的任务，在每个训练Batch中对所有任务进行平滑采样，兼顾任务的数据量进行平衡采样，同时限制数据量多的任务的上限，平滑采样可以减少任务训练有偏，在每一Batch的训练中可以减少任务任务之间的训练负迁移的情况。
        *   **分阶段训练：**

### [01:30:15 - 01:36:20] 强化学习 (Reward Model, PPO)

-   **核心论点：** 介绍ChatYuan如何通过指令微调和强化学习（RLHF）进一步提升模型性能，使其输出更符合人类偏好和安全性要求。
-   **详细展开：**
    *   **ChatYuan的产生：** 基于PromptCLUE-large模型，结合了数亿对单轮和多轮对话数据进行进一步训练，主要分为两部分：指令微调和强化学习。
    *   **RLHF三步范式 (OpenAI的ChatGPT方法):**
        1.  **Step 1: Supervised Fine-tuning (SFT) / 指令微调:**
            *   收集演示数据 (demonstration data)，即人工编写的Prompt和期望的Response。
            *   用这些数据对预训练的语言模型（Initial Language Model）进行有监督微调，使其学习到期望的输出行为。
        2.  **Step 2: Reward Model (RM) 训练:**
            *   收集对比数据 (comparison data)，即给定一个Prompt，让SFT模型生成多个不同的Response。
            *   人工标注员对这些Response进行排序，评估其好坏。
            *   使用这些排序数据训练一个奖励模型（Reward Model），该模型能够对任意Prompt-Response对进行打分，判断其质量。
            *   奖励模型通常以SFT模型为底座，训练一个分类器来判断输出是“同意”还是“拒绝”（好或坏），例如，对于“我该怎么抢劫商店”的问题，模型拒绝并告知违法是好的回答，而给出建议是不好的回答。
        3.  **Step 3: Reinforcement Learning (RL) / 强化学习:**
            *   使用PPO (Proximal Policy Optimization) 等强化学习算法，根据奖励模型对语言模型进行微调。
            *   给定一个Prompt，语言模型生成Response，奖励模型对Response打分，这个分数作为奖励信号。
            *   通过最大化奖励来更新语言模型的参数，使其生成更高质量的Response。
            *   **KL散度惩罚:** 为了防止强化学习过程中模型偏离原始SFT模型的分布太远（可能导致生成重复、无意义但得分高的内容），会引入KL散度惩罚项。这个惩罚项衡量当前策略模型与原始SFT模型输出分布的差异，限制模型在优化奖励的同时，保持生成文本的合理性和多样性。

        *   **Step 1:** Collect demonstration data, and train a supervised policy. (Initial Language Model -> Supervised Fine-tuning -> SFT Model)
        *   **Step 2:** Collect comparison data, and train a reward model. (Prompt -> Several model outputs are sampled -> A labeler ranks the outputs from best to worst -> This data is used to train our reward model -> Reward Model)
        *   **Step 3:** Optimize a policy against the reward model using reinforcement learning. (A new prompt is sampled from the prompt dataset -> This policy generates an output -> The reward model calculates a reward for this output -> This reward is used to update the policy using PPO -> Tuned Language Model (RL Policy))
