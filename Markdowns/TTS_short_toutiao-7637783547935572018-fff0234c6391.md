# 解决 Claude Code 前端设计短板：Stitch MCP 工作流

## 1. 内容概览
本视频演示了如何利用 Stitch 工具弥补 Claude Code 在前端设计方面的不足。通过简单的英语描述应用需求，Stitch 能生成完整的 UI 设计和系统，并通过 MCP (Model Context Protocol) 连接回 Claude Code，最终一键生成带有流畅动画的生产就绪代码，实现了设计与编码的无缝衔接。

## 2. 时间线

### [00:00:00 - 00:00:05] 问题提出
指出 Claude Code 擅长编码但不擅长前端设计的问题。

### [00:00:05 - 00:00:11] 解决方案：Stitch
介绍使用 Stitch 工具。用户只需简单英语描述应用并添加参考，Stitch 即可生成包含所有屏幕和设计系统的完整 UI 设计。

### [00:00:11 - 00:00:19] 导出与连接
演示在 Stitch 中满意设计后，点击 Export 选择 MCP，连接到 Claude Code，使云端完全访问设计数据。

### [00:00:19 - 00:00:24] 生成生产代码
在 Claude Code 中使用一个提示词，将设计转化为带有平滑动画的生产就绪代码，完成设计与编码的一体化。

## 3. 关键事实
*   **痛点**：Claude Code 前端设计能力弱。
*   **核心工具**：Stitch (Google 推出的 AI 设计工具)。
*   **连接协议**：MCP (Model Context Protocol)。
*   **工作流程**：自然语言描述 -> Stitch 生成 UI/Design System -> Export via MCP -> Claude Code 生成代码。
*   **优势**：避免设计与编码分离，输出带动画的高质量代码。

## 4. 可检索细节

*   **命令/配置**：`claude mcp add stitch \ --transport http ...` (视频中展示了具体的 MCP 配置代码片段)。
*   **功能描述**：Generate a full UI design with all screens and a design system; turns everything into production ready code with smooth animations.
*   **平台**：Google Stitch (网址 stich.withgoogle.com 在搜索框中出现)。
