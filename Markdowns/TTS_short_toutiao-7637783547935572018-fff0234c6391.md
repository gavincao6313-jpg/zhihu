# 解决Claude Code前端设计短板：Stitch MCP工作流

## 1. 内容概览
本视频演示了如何通过 Stitch 和 MCP (Model Context Protocol) 来解决 Claude Code 在前端设计方面的不足。通过将简单的英语描述和参考图输入 Stitch，可以生成完整的UI设计和设计系统。然后通过配置 MCP 将 Stitch 连接到 Claude Code，使得 Claude Code 能够直接访问这些设计，并将一切转化为生产就绪的代码，包括流畅的动画，从而实现了设计与编码的无缝衔接。

## 2. 时间线

### [00:00:00 - 00:00:05] 问题提出：Claude Code的局限
视频指出 Claude Code 擅长编码，但不擅长前端设计。

### [00:00:05 - 00:00:11] 解决方案：使用 Stitch 生成UI
介绍使用 Stitch 工具。用户只需缝合并用简单的英语描述应用程序，添加一些参考图，Stitch 就会生成一个完整的UI设计，包含所有屏幕和设计系统。

### [00:00:11 - 00:00:19] 连接工具：配置 MCP
当用户对设计满意后，点击导出并选择 MCP。然后连接到 Claude Code，这样 Claude Code 就能完全访问你的设计。视频中展示了 Setup MCP 的界面，包括 Client 选择 "Claude Code" 和 API key 配置。

### [00:00:19 - 00:00:24] 最终效果：一键生成生产代码
使用一个提示词，Stitch 将所有内容变成生产就绪的代码，带有流畅的动画。视频展示了生成的餐厅菜单网站示例，强调不再需要单独设计和编码。

## 3. 关键事实
*   **工具组合**: Claude Code + Stitch + MCP (Model Context Protocol)。
*   **痛点**: Claude Code 不擅长前端设计。
*   **Stitch 功能**: 
    *   接收简单的英语应用描述。
    *   接收参考图 (references)。
    *   生成完整的 UI 设计，包含所有屏幕 (all screens) 和设计系统 (design system)。
*   **MCP 作用**: 作为桥梁，将 Stitch 生成的设计连接到 Claude Code，赋予 Claude Code 对设计的完全访问权限。
*   **最终输出**: 生产就绪的代码 (production ready code)，包含流畅的动画 (smooth animations)。
*   **优势**: 无需单独设计和编码，一体化流程。

## 4. 可检索细节

*   **Setup MCP 界面**: 显示 Client 为 "Claude Code"，需要配置 API key 和 Configuration (包含 `claude mcp add stitch ...` 命令)。
*   **输出示例**: Craving 餐厅网站，包含 "Make a Moment with Craving" 标题，菜单分类 (Appetizers, Mains, Desserts, Drinks)。
