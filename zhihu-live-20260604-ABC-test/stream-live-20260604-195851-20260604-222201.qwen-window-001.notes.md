## Faithful Notes
本窗口为AI产品经理课程关于“一人公司（OPC）”的完整方法论讲解，涵盖法律风险、产品验证、商业模式及思维层级。

**1. OPC定义与法律红线**
- **核心定义：** OPC不是法律意义上的“个人独资企业”，而是经营模式上的“一人公司”。即一个核心经营者利用AI/自动化工具/外包，独立完成从产品设计、开发、获客、交付到变现的全流程。
- **致命法律风险：** 严禁注册“个人独资公司”（只有一个自然人股东）。此类公司为无限责任，若注册资本10万但被索赔200万，个人需承担无限连带责任。
- **合规操作：** 必须找另一个人（如父母，非夫妻关系）占股1%，使其成为“有限责任公司”，责任仅限于注册资本金。
- **与副业区别：** 自由职业者（Freelancer）出卖时间/技能，不可规模化；传统个体户依赖线下人力；OPC通过系统放大能力，可规模化、有产品、有流程。

**2. 实战案例：AI课程辅助工具**
- **痛点发现：** 直播课中用户知识背景不一致，基础问题（Token/RAG是什么）打断进度，不讲又影响后续理解。
- **解决方案思路：** 假设“无限成本”下的最优解是“1V1授课+超强助教”。在OPC逻辑下，实时1V1成本高且小模型效果差，故收敛为“AI助教”Demo。
- **Prompt工程：** 讲师现场展示了用于生成该工具的Prompt，强调“大道至简”，核心价值链为：听->转写->命中知识图谱->划词解释->笔记->课后小结。
- **MVP原则：** 只做市面上没有的“唯一功能”，放弃大而全。Demo阶段不做注册登录、社区、打卡、成就系统。公测版才加注册登录。
- **技术纠结：** ASR识别精度不够，高精度ASR价格贵6倍，目前正尝试用Router配置等高端策略解决成本问题。

**3. 需求验证与反馈信号**
- **无效反馈：** 用户说“这个不错”通常是客气，价值不大。
- **有效反馈：**
    - 负面情绪：“怎么还收费？”、“讨厌”。（说明有购买意愿，只是对价格敏感，“谈价格的才是买主”）
    - 最大正面反馈：直接交钱，“别废话，拿钱走人”。
- **收费模式：** OPC初/中期必须用户端收费（To C/To B均可），严禁免费+广告模式。原因：OPC需规模小、深度深、回款快，广告业态复杂度高且吸引羊毛党。

**4. 从PMF到MPF的范式转移**
- **PMF局限：** Product-Market Fit只是商业闭环的1/4（设计开发环节）。仅有PMF可能导致“拿着锤子找钉子”，做了很多满足需求的产品但无法变现。
- **MPF模型：** Market（市场需求）+ Product（产品服务）+ Founder（创始人/主体）。
- **核心变化：** 核心不是P（产品），而是F（Founder）。你能找到的市场才是市场，你能提供的产品才是产品。因为是你，所以才能交易。
- **OPC三要素：** 轻（模式轻、无重资产、无雇佣）、快（回款<1个月、Demo<3天）、小（商业闭环简单，一手交钱一手交货）。
- **避坑指南：** 不要考虑腾讯/大厂怎么做；不要做复杂闭环（如爱奇艺模式）；不要做App Store（推广难、审核慢），首选小程序或Web。

**5. 启动策略：先M后P**
- **赛博摆摊理论：** OPC本质是赛博摆摊。第一步不是决定卖什么（P），而是决定在哪摆摊（M/流量/渠道）。
- **场景决定产品：** 学校门口卖萝卜刀，别墅区卖水晶手串（卖惨逻辑），网红街卖美妆蛋。先有流量结构，再有产品匹配。
- **冷启动资源：**
    - 前领导/前同事：了解旧公司痛点，用Demo换合作。
    - 大学学工部：针对就业/心理健康痛点，捐赠Token换取案例/背书。
    - 反向定制：像Oracle一样，先找头部客户定制，再模块化复制。
- **海外套利机会：** 关注美国App Store增长率>50%、收入/下载>$0.5、评分<4分或更新慢的产品。做差异化钩子，强力变现（甚至付费后功能空白页在国外也可行，利用遗忘率）。

**6. 思维层级与微笑曲线**
- **六层思维：** 技术思维 -> 工程思维 -> 产品思维 -> 市场思维 -> 商业思维 -> 资本思维。
- **OPC定位：** 对应微笑曲线两端（研发/品牌），避免中间组装环节。目标是让自己变得更值钱，而不仅是多挣钱。
