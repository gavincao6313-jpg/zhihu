# 运行日志总账

Date: 2026-05-17

本文档用于记录每日任务运行情况、代码修改前后的运行对比、异常日志处理过程、验证结果和剩余风险。

它是项目共享文件，Windows 和 Mac 都需要遵循同一记录规则。原始 `*.log` 保持本地，不直接提交；需要共享的证据摘录、分析结论和复盘结果写入本文档或 `runs/*.md`。

## 记录原则

1. 每天至少按日期新增一节。
2. 每次生产运行、验证运行、异常中断、重跑，都要留下可回溯记录。
3. 异常日志不能只贴现象，要写清楚判断依据、处理动作、验证结果和是否复发。
4. 涉及代码修改前后对比时，要记录修改前基线、修改后结果和对比结论。
5. 不记录 API key、Cookie、Authorization、签名 URL、完整本机私有路径等敏感信息。

### 异常日志处理

#### 异常 1：原始 `zhihuTTS.log` 曾被 Git 跟踪

- 现象：虽然 `.gitignore` 已有 `*.log`，但历史上 `zhihuTTS.log` 已进入 Git 跟踪。
- 初步判断：这是历史跟踪遗留问题，后续 `.gitignore` 不能自动让已跟踪文件退出版本控制。
- 处理动作：从 Git 跟踪中移除 `zhihuTTS.log`，保留本机文件；共享证据改为写入 `runs/*.md` 或本文档。
- 验证方式：检查 `git ls-files zhihuTTS.log`。
- 验证结果：远端 `main` 的 `ea77639` 后不再跟踪该文件。
- 后续风险：如果手工强制添加 `*.log`，仍可能误提交；已通过 `githooks/pre-commit` 拦截。

#### 异常 2：OpenWolf 本机遥测文件曾被 Git 跟踪

- 现象：`.wolf/hooks/_session.json`、`.wolf/token-ledger.json` 等文件包含本机路径、会话和工具运行状态。
- 初步判断：这些文件属于本机工具遥测，不是项目共识，不应共享给对端。
- 处理动作：从 Git 跟踪中移除，并加入 `.gitignore`。
- 验证方式：检查 `git ls-files` 和 pre-commit 规则。
- 验证结果：已不再跟踪，新增同类文件会被 hook 拦截。
- 后续风险：`.wolf/memory.md`、`.wolf/buglog.json` 仍可能混入局部路径；后续记录时要主动脱敏和压缩为项目事实。
