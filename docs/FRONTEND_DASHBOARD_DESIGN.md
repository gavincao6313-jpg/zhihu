# 直播转写控制台 — 前端设计需求

> **创建**: 2026-06-02
> **状态**: 待设计（计划 2026-06-03 启动）
> **背景**: 当前工具始终是后端 CLI，操作全靠 BAT 文件和命令行，没有可视化界面，无法形成稳定可复用的操作 SOP，也无法让非技术用户独立操作。

---

## 一、目标

把整个直播转写流程从"后端工具"变成"完整软件产品"。

操作者应该能：
1. 一键启动直播转写（不需要了解任何命令行细节）
2. 实时看到流水线运行状态（哪个 chunk 在处理，有无错误）
3. 直播结束后在同一界面查看所有产物（逐字稿、关键帧、QC、最终 MD）
4. 管理历史直播记录（翻看过去每场直播的完整产物）

## 三、核心页面清单

### 3.1 启动页
- URL 输入框（支持粘贴含 `&` 的完整知乎 URL）
- 模式选择（Gemini / Qwen / 仅转录）
- 认证状态显示（Cookie 剩余有效时长，到期警告）
- [重新登录] 按钮

### 3.2 运行监控页（直播进行中）
- 当前 base name / 直播标题
- 实时运行时长
- Chunk 进度条（已完成/总量未知）
- 最近 N 条 chunk 卡片，每张显示：时间段、字符数、帧数、状态（✅/⚠️/❌）
- 实时日志滚动窗口（tail 模式）
- QC 警告横幅（出现 warning 时高亮）
- [手动停止] 按钮

### 3.3 直播详情页（产物查看）

**Tab 1 — 概览**
- source_status（full / partial / interrupted）
- 总时长 / chunk 数 / transcript 字数 / 帧数 / gap 数
- QC 警告列表
- body_transcript_ratio / fact_retention / narrative_retention

**Tab 2 — 逐字稿**
- combined-transcript 全文，时间戳锚点可跳转
- 搜索框

**Tab 3 — 关键帧**
- 按时间轴排列的帧缩略图网格
- 点击放大，显示帧类型（slide/annotation/context/speaker）和时间戳

**Tab 4 — QC 记录**
- final-qc.json 可视化（每条警告展开显示详情）
- Gemini vs Qwen 对比面板（双模型运行时）

**Tab 5 — 最终产物**
- Gemini NotebookLM MD 预览（Markdown 渲染）
- Qwen NotebookLM MD 预览
- [下载] 按钮

### 3.4 历史记录页
- 所有历史直播列表（从 `runs/` 目录扫描）
- 每条记录：日期 / 标题 / 时长 / chunk 数 / QC 状态（full/partial/警告数）
- 点击进入 3.3 详情页

### 3.5 设置页
- 认证管理（查看 Cookie 有效期 / 重新登录）
- API Key 配置（GEMINI_API_KEY / DASHSCOPE_API_KEY）
- venv Python 路径
- 默认合成模式

## 四、技术选型方向（待讨论）

| 方向 | 优势 | 劣势 |
|------|------|------|
| **Streamlit（纯 Python）** | 极低开发成本，Python 原生，与现有代码无缝集成 | UI 定制度低，不够产品化 |
| **FastAPI 后端 + React 前端** | 最佳 UI 体验，清晰前后端分离 | 开发量大 |
| **Electron（本地桌面应用）** | 无需 Web 服务，直接调用本地 Python | 跨平台打包复杂 |
| **Next.js + FastAPI** | 可远程访问，生产级 | 最高开发成本 |

> **初步推荐**: Streamlit 快速验证 MVP → 确认功能后迁移到 FastAPI + React 正式版。

## 五、后端需新增的接口能力

当前后端全部是 CLI 工具，需新增：

| 接口 | 说明 |
|------|------|
| `GET /api/runs` | 扫描 `runs/` 目录，返回历史直播列表 |
| `GET /api/runs/{base}` | 返回某场直播的 manifest、QC、chunks 汇总 |
| `GET /api/runs/{base}/transcript` | 返回 combined-transcript 全文 |
| `GET /api/runs/{base}/frames` | 返回关键帧列表（路径 + 元数据） |
| `GET /api/runs/{base}/markdown` | 返回 Markdowns/ 中的最终 MD 内容 |
| `POST /api/launch` | 启动直播转写（调用 `zhihuTTS_stream.py`） |
| `GET /api/status/{run_id}` | 实时查询当前运行状态（SSE 或 WebSocket） |
| `POST /api/synthesize` | 对已有 chunks 触发 Gemini/Qwen 合成 |
